Synthetic Sciences ships two products that work independently and pair naturally.

**[Thesis](/introduction)** is a graph-based research workspace. Hypotheses, experiments, sources, and decisions live as durable, branchable nodes. Ships an 86-tool MCP server for Cursor, Claude Code, Codex, and others.

**[Synsci CLI](/cli/introduction)** is a terminal agent for scientific work. Research modes for chemistry, biology, physics, materials, machine learning, and mathematics. Bundled domain tools, native cloud sync, and included credits.

Both share a Synthetic Sciences account, and Synsci CLI can read and write Thesis state through MCP.

## Pick a starting point

- **[Set up Thesis](/quickstart).** Sign in, create your first research node, index a source, and connect an agent host.
- **[Install Synsci CLI](/cli/installation).** Run `npx synsci` once. The all-in-one installer drops the binary, authenticates you, and pre-fetches credentials. Then `synsc web`.
- **[Connect Thesis to Cursor / Claude Code](/mcp/quickstart).** `npx --yes @synsci/thesis setup` writes the MCP config for every supported host.
- **[Pair Thesis + Synsci CLI](/cli/mcp).** Wire Synsci CLI to Thesis as an MCP host so the agent reads and writes your research graph.

## Which product solves what

| Job | Reach for |
|---|---|
| Track hypotheses, experiments, decisions, and dead ends across a research program | [Thesis](/introduction) |
| Run experiments under an explicit compute approval gate | [Thesis compute](/concepts/compute) |
| Search prior work alongside indexed papers and repos | [Thesis knowledge layer](/concepts/knowledge-layer) |
| Run DFT, molecular dynamics, or other compute work in chemistry | [Chemistry](/chemistry/introduction) |
| Predict protein structures, run sequencing pipelines, do single-cell analysis | [Biology](/biology/introduction) |
| Set up CFD, lattice QCD, or quantum simulations | [Physics](/physics/introduction) |
| Model crystal structures, run high-throughput materials screening | [Materials](/materials/introduction) |
| Train, fine-tune, evaluate, or serve ML models | [Machine learning](/ml/introduction) |
| Symbolic computation, formal proofs, optimization | [Mathematics](/math/introduction) |
| Sync HF, W&B, Modal, Lambda, AWS, GCP credentials automatically | [Synsci CLI credentials](/cli/credentials) |

## Common workflows

<AccordionGroup>
  <Accordion title="Research-only. Thesis on its own">
    Start at [Thesis quickstart](/quickstart). Create a root node, index a paper or two, talk to the agent, stage an experiment when you're ready. The web app at [thesis.syntheticsciences.ai](https://thesis.syntheticsciences.ai) is enough. You don't need any CLI.
  </Accordion>

  <Accordion title="Coding-only. Synsci CLI on its own">
    Install with `npx synsci`, then run `synsc web --mode <your-domain>` to open the browser UI. The bundled domain tools and credential sync mean you can start running workflows in minutes. No Thesis account needed.
  </Accordion>

  <Accordion title="Both. Synsci CLI runs the work, Thesis tracks the program">
    Set up both products. Connect them with `synsc mcp add thesis --url https://thesis.syntheticsciences.ai/mcp-server` so the agent can read and write your research graph. The agent stages experiments in Thesis while it runs the actual work in your repo. See [Connect Synsci CLI to Thesis](/cli/mcp).
  </Accordion>

  <Accordion title="Cursor or Claude Code + Thesis (no Synsci CLI)">
    Use your existing IDE agent host. Run `npx --yes @synsci/thesis setup` to wire Thesis into the host's MCP config. Your agent gets the 86 Thesis tools alongside its built-ins. Synsci CLI isn't required.
  </Accordion>
</AccordionGroup>

## Account and billing

Both products share a Synthetic Sciences account. Sign in once and:

- Thesis web app and MCP server use your account for project access.
- Synsci CLI dashboard at [cli.syntheticsciences.ai](https://cli.syntheticsciences.ai) issues Bearer tokens, holds synced credentials, and tracks credit usage.

Plans and credits are scoped per product. You can subscribe to one without the other, or both with a combined plan. Visit your account page for plan details.
