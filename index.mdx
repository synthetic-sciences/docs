A graph-based research workspace and a terminal agent for scientific work. Two products that ship independently and pair through MCP.

## Get going in one command

Install the CLI:

```bash
npx synsci
```

Or open the research workspace at [thesis.syntheticsciences.ai](https://thesis.syntheticsciences.ai). Both share a Synthetic Sciences account.

## Thesis

A graph-based research workspace. Hypotheses, experiments, sources, and decisions live as durable, branchable nodes. Ships an 86-tool MCP server for Cursor, Claude Code, Codex, and other agent hosts.

- **[Introduction](/introduction).** What Thesis is for and how it works.
- **[Quickstart](/quickstart).** Sign in, create your first node, index a source, talk to the agent.
- **[Web app tour](/web-app).** Every UI area explained.

## Thesis MCP

Connect Thesis to the agent host you already use. The setup helper writes the MCP config for every detected host in one command.

- **[MCP quickstart](/mcp/quickstart).** `npx --yes @synsci/thesis setup`.
- **[Supported hosts](/mcp/supported-hosts).** Claude Code, Cursor, Codex, OpenCode, and more.
- **[MCP tools reference](/mcp/tools-overview).** All 86 tools by category.

## Synsci CLI

A terminal agent for scientific work. The binary is `synsc`. Pick a research mode and the agent loads the right domain conventions, tools, and backends for the work.

- **[Introduction](/cli/introduction).** What `synsc` is and what each research mode covers.
- **[Installation](/cli/installation).** `npx synsci` does the entire setup.
- **[Quickstart](/cli/quickstart).** Connect, pick a mode, send your first prompt.

### Research modes

| Mode | For |
|---|---|
| [Research](/cli/research/research) | Cross-disciplinary work, literature review, hypothesis generation, GPU compute orchestration |
| [Physics](/cli/research/physics) | Simulation, PDE solving, dynamical systems, symbolic regression |
| [Chemistry](/cli/research/chemistry) | Drug discovery, molecular design, ADMET, docking, structure prediction |
| [Biology](/cli/research/biology) | Bioinformatics, sequencing pipelines, single-cell analysis, structure prediction |
| [Write](/cli/research/write) | LaTeX papers, grant proposals, literature reviews with verified citations |
| [Flywheel](/cli/research/flywheel) | Build a specialized LLM from your production data |

## Common workflows

<AccordionGroup>
  <Accordion title="Research-only. Thesis on its own.">
    Start at the [Thesis quickstart](/quickstart). Create a root node, index a paper or two, talk to the agent, stage an experiment when you're ready. The web app at [thesis.syntheticsciences.ai](https://thesis.syntheticsciences.ai) is enough. You don't need any CLI.
  </Accordion>

  <Accordion title="Coding-only. Synsci CLI on its own.">
    Install with `npx synsci`, then run `synsc web --mode <your-domain>`. The bundled domain tools and credential sync mean you can start running scientific workflows in minutes. No Thesis account needed.
  </Accordion>

  <Accordion title="Both. Synsci CLI runs the work, Thesis tracks the program.">
    Set up both products. Connect them with `synsc mcp add thesis --url https://thesis.syntheticsciences.ai/mcp-server`. The agent stages experiments in Thesis while it runs the actual work in your repo. See [Connect Synsci CLI to Thesis](/cli/mcp).
  </Accordion>

  <Accordion title="Cursor or Claude Code with Thesis.">
    Use your existing IDE agent host. Run `npx --yes @synsci/thesis setup` to wire Thesis into the host's MCP config. Your agent gets the 86 Thesis tools alongside its built-ins. Synsci CLI is optional.
  </Accordion>
</AccordionGroup>

## Account and billing

Both products share a Synthetic Sciences account. Thesis web app and MCP server use your account for project access. Synsci CLI dashboard at [cli.syntheticsciences.ai](https://cli.syntheticsciences.ai) issues Bearer tokens, holds synced credentials, and tracks credit usage.

Plans and credits are scoped per product. Subscribe to either or both. Visit your account page for plan details.
