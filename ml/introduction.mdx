Synsci handles machine learning workflows from data prep through training, evaluation, and deployment. Activate the ML mode and the agent reasons about your prompts the way an ML engineer would: it picks the right framework, the right hardware, and the right scaling strategy.

## Activate the ML mode

```bash
synsc web --mode ml
```

For a one-shot run:

```bash
synsc run --mode ml "Fine-tune llama-3-8b with QLoRA on this dataset, log to wandb, and serve with vLLM"
```

To set ML as the default for your current working directory:

```bash
synsc mode set ml
```

## Who this is for

- ML engineers and researchers training, fine-tuning, and evaluating models
- Inference and infrastructure teams running production model serving
- Post-training and alignment researchers using RLHF and DPO
- Eval teams running benchmark harnesses
- Data engineers building dataset pipelines for training

## What you can do

- Distributed training across GPUs and nodes
- LoRA, QLoRA, ReFT, and full-parameter fine-tuning
- RLHF, DPO, GRPO, KTO, and other post-training methods
- Multi-backend inference (vLLM, TensorRT-LLM, SGLang, llama.cpp)
- Evaluation against MMLU, BIG-bench, HELM, and custom suites
- Experiment tracking, sweeps, and artifact management

## Frameworks the agent uses

When you describe an ML task, the agent reaches for these natively. You don't need to invoke them by name.

| Category | Stack |
|---|---|
| Training frameworks | PyTorch DDP, FSDP, DeepSpeed, Megatron, Axolotl |
| Post-training | TRL, OpenRLHF, GRPO, PPO, DPO, RLOO, KTO |
| Inference and serving | vLLM, TensorRT-LLM, SGLang, llama.cpp, TGI |
| Evaluation | lm-eval-harness, BIG-bench, HELM |
| Fine-tuning | LoRA, QLoRA, ReFT, full SFT |
| Distributed and infra | Ray, SkyPilot, Slurm, K8s job templates |
| Hubs and tracking | Hugging Face Hub, Weights & Biases, MLflow |
