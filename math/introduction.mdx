Synsci handles mathematical workflows from symbolic computation through formal proofs and large-scale optimization. Activate the math mode and the agent reasons about your prompts the way a mathematician or applied mathematician would: it picks the right tool for symbolic, numeric, or formal work.

## Activate the math mode

```bash
synsc web --mode math
```

For a one-shot run:

```bash
synsc run --mode math "Formalize this proof in Lean 4 and check it compiles"
```

To set math as the default for your current working directory:

```bash
synsc mode set math
```

## Who this is for

- Pure mathematicians using proof assistants for formal verification
- Applied mathematicians doing symbolic algebra and analysis
- Optimization researchers building convex and integer programming models
- Numerical analysts validating discretizations and methods
- Computer algebra researchers using high-precision and exact arithmetic

## What you can do

- Symbolic algebra and calculus
- Formal theorem proving and proof assistant work
- Convex, integer, and combinatorial optimization
- Numerical analysis and scientific computing
- High-precision arithmetic and computer algebra
- Algorithmic mathematics and complexity analysis

## Frameworks the agent uses

When you describe a math task, the agent reaches for these natively. You don't need to invoke them by name.

| Category | Stack |
|---|---|
| Symbolic computation | SymPy, SageMath |
| Theorem proving | Lean 4 with mathlib4, Coq, Isabelle/HOL |
| Numerical analysis | NumPy, SciPy, JAX |
| Convex optimization | CVXPY, MOSEK, ECOS |
| Integer programming | JuMP, Gurobi, COIN-OR |
| High-precision | mpmath, gmpy2, FLINT |
| Computer algebra | SageMath, Singular, Macaulay2 |
