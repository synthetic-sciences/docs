Thesis works out of the box without any external accounts or API keys. The core workspace, research graphs, nodes, artifacts, the research log, and MCP tools, runs entirely on its own. External integrations extend Thesis with additional capabilities: indexing papers and repositories, powering the AI agent loop, enabling GPU compute for sub-agents, and managing subscriptions. You connect each integration from **Settings → API Keys** in the Thesis web app.

## What each integration enables

- **Nia.** Powers external knowledge: index papers, repositories, documentation sites, and datasets. Enables search, read, grep, explore, and deep research across those sources.
- **Exa.** Powers the **Deep Research** pipeline. When you trigger a deep research job, Thesis uses Exa to search and retrieve full text from research papers and web sources, then synthesizes a report.
- **OpenAI.** Powers graph embeddings. Thesis embeds your committed graph content into a local knowledge index so the agent and search can retrieve relevant nodes alongside external sources.
- **GitHub.** Powers **Tracer** code search and linked repository access. Connect your GitHub account to search across public and private repositories and browse repository file trees directly in Thesis.
- **Anthropic.** Powers the main Thesis agent. The agent uses Claude to run its ReAct loop, reading your graph, searching sources, staging nodes, writing log entries, and coordinating sub-agents.
- **Stripe.** Powers subscriptions and credits. Stripe handles plan checkout, upgrades, billing portal access, and usage-based credit tracking for knowledge and compute spending.
- **Modal.** Powers managed sandbox runners for sub-agents. When the agent spawns a sub-agent after your approval, Modal provides the isolated GPU environment where the agent executes.
- **Lambda Cloud.** Powers GPU instance leasing. Lambda Cloud lets you acquire on-demand GPU machines (A10G, A100, H100) for compute-intensive research tasks.


## How integrations are grouped

Thesis integrations fall into four functional areas:

| Area | Services | What you unlock |
|---|---|---|
| **Knowledge** | Nia, Exa, OpenAI, GitHub | External source indexing, Deep Research, graph embeddings, code search |
| **AI** | Anthropic | Agent chat, ReAct loop, session compression, deep-research synthesis |
| **Compute** | Modal, Lambda Cloud | Sub-agent sandbox runners, GPU instance leasing |
| **Billing** | Stripe | Subscriptions, plan upgrades, credit purchases, billing portal |

## Connecting an integration

All integrations you manage as a user are connected through **Settings → API Keys**. Enter your API key for each service you want to enable. Thesis stores and uses the key on your behalf, you never need to edit configuration files or set environment variables.

<Note>
  If you are self-hosting Thesis, some integrations (such as Stripe and compute providers) may require additional server-side configuration by your administrator. This page covers what you control as a user.
</Note>

## What happens without each integration

Thesis degrades gracefully when an integration is not connected. Nothing breaks, features that depend on the missing service simply become unavailable.

| Integration | Without it |
|---|---|
| **Nia** | External source indexing and search are unavailable. You can still search your own graph content. |
| **Exa** | Deep Research jobs cannot run. The agent will report that Deep Research requires Exa to be configured. |
| **OpenAI** | Graph embedding and graph-as-knowledge-base retrieval are disabled. Knowledge search falls back to Nia results only. |
| **GitHub** | Tracer code search is unavailable. Public GitHub calls may still work at reduced rate limits. Linking private repositories requires connecting your account. |
| **Anthropic** | The agent chat interface returns a configuration message instead of running. Deep Research synthesis is skipped, if Exa is configured, raw source summaries are still written to your project. |
| **Stripe** | Billing, plan upgrades, and credit purchases are unavailable. |
| **Modal** | Sub-agent sandbox runners cannot be started. The approval flow still works but launching a sandboxed agent requires Modal to be configured. |
| **Lambda Cloud** | Lambda GPU instances cannot be leased. |

<Tip>
  You do not need every integration. Connect only what your workflow requires. For example, if you are using Thesis purely for graph organization and export, you may not need any external services at all.
</Tip>

## Knowledge integrations in depth

Nia, Exa, and OpenAI each cover a different part of the knowledge layer, they are not redundant.

- **Nia** handles *indexed sources*: external papers, repositories, docs, and datasets you explicitly add to your knowledge base. The agent and search tools query these directly.
- **Exa** handles *discovery*: when you run a Deep Research job, Thesis searches Exa for relevant papers and web content on the fly, fetches their full text, and synthesizes a report using Anthropic.
- **OpenAI** handles *graph embeddings*: your own Thesis graph content is embedded so it participates in unified knowledge search alongside Nia results.

All three can be active at the same time. Thesis merges and deduplicates results from graph embeddings and Nia, then presents a unified ranked list.

## GitHub: two levels of access

GitHub integration works at two levels:

1. **Server-level token**, enables Tracer code search across public repositories for all users.
2. **Per-user OAuth connection**, connects your personal GitHub account so Thesis can access your private repositories and linked repo file trees.

You connect your personal GitHub account from **Settings → Connected Accounts**. Private repository access uses your own authorization and does not expose your token to other users.
