Synsci handles materials science workflows from DFT through high-throughput screening and ML interatomic potentials. Activate the materials mode and the agent reasons about your prompts the way a materials scientist would: it picks the right level of theory, the right code, and the right approach for the system size.

## Activate the materials mode

```bash
synsc web --mode materials
```

For a one-shot run:

```bash
synsc run --mode materials "Relax this perovskite supercell with MACE, then compute phonons with phonopy"
```

To set materials as the default for your current working directory:

```bash
synsc mode set materials
```

## Who this is for

- Computational materials scientists running DFT and post-DFT calculations
- Battery, catalysis, and electronic materials researchers
- High-throughput screening teams searching chemical and structural space
- Crystallography and structure prediction researchers
- ML potential developers training and validating models

## What you can do

- DFT total energy, band structure, and electronic property calculations
- Phonons, elastic constants, and thermodynamic properties
- Ab initio molecular dynamics and ML-driven dynamics
- Crystal structure prediction and high-throughput screening
- Building and validating ML interatomic potentials
- Phase diagrams and CALPHAD modeling

## Frameworks the agent uses

When you describe a materials task, the agent reaches for these natively. You don't need to invoke them by name.

| Category | Stack |
|---|---|
| DFT for materials | VASP, Quantum ESPRESSO, GPAW, ABINIT, CASTEP |
| ML interatomic potentials | MACE, Allegro, NequIP, M3GNet, CHGNet |
| Structure prediction | CALYPSO, USPEX, AIRSS, PyXtal |
| Phonons and thermodynamics | phonopy, phono3py, alamode |
| High-throughput | pymatgen, atomate, jarvis-tools, ase |
| CALPHAD | pycalphad, OpenCalphad |
| Visualization | OVITO, VESTA, ASE |
