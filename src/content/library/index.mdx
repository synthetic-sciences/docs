**Atlas Library** is the knowledge-source product in the Atlas suite. It is Atlas's library service: index external material once, then search it, ask grounded questions, and pull the useful answers straight into the [research graph](/atlas/index) as evidence.

The library uses your Atlas authentication only. You do not configure any separate search key.

## Source types

| Type | Material |
| --- | --- |
| `repository` | A GitHub or Git repository. |
| `documentation` | A docs site or technical docs. |
| `research_paper` | A paper URL or PDF. Use `research_paper`, never `paper`. |
| `huggingface_dataset` | A dataset page. |
| `local_folder` | A local folder, walked and uploaded. **Private by default.** |

Local folders are the newest source type: index a working directory privately without pushing it anywhere public. See [Index sources](/library/indexing).

## The flow

```text
library:add  →  library:search / library:ask  →  save into the graph
 (index)         (universal | targeted | web | deep)   (--save-to / --attach)
```

1. **Index** material with `library:add` (a URL, or a local folder with `--path`).
2. **Search** with `library:search --mode …` or **ask** with `library:ask`.
3. **Ground it** into the graph: `library:search --save-to <node>` saves hits as evidence, and `library:ask --attach <node>` attaches the grounded answer.

## Index your first source

```bash
atlas library:add --type research_paper --url https://arxiv.org/abs/1706.03762
atlas library:list --format=json
atlas library:search --query "windowed attention"
```
