This page is grounded in the `@synsci/atlas@0.5.11` command registry and bundled skill package. It is the high-level map; [Commands](/commands) is the exact route table.

## Package surface

| Surface | What ships |
| --- | --- |
| CLI binary | `atlas` from `@synsci/atlas` |
| Runtime | `src/atlas-runtime/` with command registry, config, dispatch, HTTP client, local commands, output, and parsing |
| Skills | `8` bundled skills under `skills/` |
| API base | `https://app.syntheticsciences.ai/api/v1` by default |
| Commands | `106` public commands |
| Node requirement | Node `18+` |

## Local commands

These commands run local CLI behavior:

| Command | Local behavior |
| --- | --- |
| `login` | Starts a localhost callback, opens browser approval, redeems a one-time token, and writes the API key to local config. |
| `logout` | Removes credentials for the active profile. |
| `doctor` | Checks config path, active profile, API base URL, backend status, auth status when a key exists, and bundled skill presence. |

`whoami` is not local; it calls `/auth/status`.

## Map primitives

Atlas map work is built from:

- `nodes:create` for new durable map state.
- `nodes:save` for changes to existing nodes.
- `nodes:branch`, `link:add`, and `link:remove` for topology.
- `map:view`, `link:parents`, `link:children`, and render commands for reads.
- `access:*` for access policy.
- `labels:*` for root-scoped labels.
- `draft:*` for multi-agent edit coordination.
- `evidence:link/refresh` for public HTTPS JSON evidence attached by URL.
- `runs:*` for node-scoped run records.

Use [Nodes and maps](/graph) for the concepts and [Build a first graph](/first-graph) for a workflow.

## Evidence primitives

Evidence files are attached to nodes. The CLI supports:

- A high-level one-shot upload with `evidence:add`.
- Lower-level prepare/finalize upload calls.
- Evidence listing, reads, notes, and deletion.
- Cross-node file listing with `evidence:files`.

The one-shot upload path validates local files, prepares upload records, writes bytes to the signed upload URL, then finalizes the batch.

## Library and research primitives

Atlas library and research commands cover:

- `library:list`, `library:add`, `library:show`, `library:update`, `library:delete`, and `library:sync`.
- `library:tree`, `library:read`, and `library:grep` for source inspection.
- `library:search` for unified search.
- `library:ask` for synchronous document Q&A.
- `library:runs:*` for async library jobs.
- `research:ask` and `research:runs:*` for broader research tasks.
- `usage:summary` for library and research usage.

Supported source types are `repository`, `documentation`, `research_paper`, `huggingface_dataset`, `local_folder`, `slack`, `google_drive`, and `connector`. Atlas also normalizes common aliases like `repo`, `docs`, `dataset`, and `paper`; prefer the stable full names in scripts.

## Output and safety

The CLI supports `json`, `tsv`, `csv`, `table`, and `pretty` output. Use `--format=json` for agents.

Destructive commands are safety-gated and require `--yes` or `--force`. This includes deletes, revokes, cancellations, disconnects, and key rotation.

Use `--idempotency-key` when retrying writes from an external orchestrator.
