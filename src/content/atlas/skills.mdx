Atlas skills ship inside `@synsci/atlas`. Agents should discover them from the installed package and should not install a separate Atlas skill bundle.

The skills are workflow instructions, not extra runtime binaries. They tell an agent how to use the Atlas CLI safely for a specific job.

| Skill | Use |
| --- | --- |
| `atlas` | General Atlas guidance, command discovery, and map workflow setup. |
| `atlas-auto` | Autonomous map continuation with explicit budget and durable state. |
| `atlas-lookahead` | Planning-only frontier expansion before runs. |
| `atlas-prove` | Local formalization and proof workflow scaffolding. |
| `atlas-reproduce` | Source-backed claim validation with evidence and map saves. |
| `atlas-search` | Search indexed library sources and turn grounded results into map context. |
| `atlas-to-graph` | Port source material into nodes, evidence, and durable links. |
| `atlas-tree` | Render map structure as terminal-first tree views. |

## Pick the right skill

| User request | Use |
| --- | --- |
| "Set up Atlas" or "show me the commands" | `atlas` |
| "Turn this paper/repo/source into map state" | `atlas-to-graph` |
| "Search my indexed library sources" | `atlas-search` |
| "Plan next steps without running them" | `atlas-lookahead` |
| "Continue this frontier autonomously" | `atlas-auto` |
| "Reproduce or audit this claim" | `atlas-reproduce` |
| "Track a proof attempt" | `atlas-prove` |
| "Render the map/tree" | `atlas-tree` |

## Verify skills

```bash
atlas doctor --format=json
atlas help --format=json
```

`doctor` checks the package skill directory and reports the count and names.
