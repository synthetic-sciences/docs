Atlas ships nine skills inside `@synsci/atlas`. They are workflow instructions that teach a coding agent how to drive Atlas safely; they are not extra binaries. Run `atlas install` to drop them into the agents on your machine. The router (`atlas`) â€” which also folds in the read-only map commands (`map:tree`, `map:lineage`) â€” is covered in the [CLI overview](/atlas/cli-overview); the library skill (`atlas-search`) lives under [Atlas Library](/library/skills). The rest drive the graph.

## Graph skills

| Skill | Trigger / use case | First command |
| --- | --- | --- |
| `atlas-lab` | Doing or continuing actual research in a project, and capturing finished runs. Runs the loop: brief â†’ hypothesis â†’ plan â†’ run â†’ compare â†’ decision. | `atlas brief --project <id>` |

| `atlas-map` | Turning papers, repos, docs, or notes into durable map nodes and evidence, without launching runs. | `atlas node:create` / `atlas library:tree` |

## Pick the right one

| User request | Skill |
| --- | --- |
| "Continue the research in this project" / "record this run" | `atlas-lab` |
| "Plan next steps" or "run this frontier autonomously" | `atlas-frontier` |
| "Turn this paper/repo into map state" | `atlas-map` |
| "Optimize this prompt/config against a metric" | `atlas-optimize` |
| "Run the whole campaign from a goal and verify it" | `atlas-autoresearch` |
| "Reproduce this paper" / "does this result hold?" | `atlas-reproduce` |
| "Write this up as a paper" | `atlas-paper` |

## Verify

```bash
atlas doctor --format=json   # reports the skill count and names
atlas install --dry-run      # preview where each skill would land
```

`doctor` checks that all nine skills are present in the installed package. Skills are bundled in the CLI â€” never configure separate skill downloads.
