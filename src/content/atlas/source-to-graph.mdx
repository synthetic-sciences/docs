---
title: "Turn library sources into map state"
description: "Index external material, ask grounded questions, and save useful answers."
icon: "search"
---

Library sources give Atlas searchable context. Map nodes turn that context into durable research state. The usual flow is: index, inspect, search, ask, save.

## Choose the source type

| Material | Use this type |
| --- | --- |
| GitHub or Git repository | `repository` |
| Product docs or technical docs | `documentation` |
| Paper URL or PDF | `research_paper` |
| Dataset page | `huggingface_dataset` |
| Local folder source | `local_folder` |
| Slack source | `slack` |
| Google Drive source | `google_drive` |
| Generic connector source | `connector` |

Use `research_paper` for papers. Do not use `paper`.

## Index the source

```bash
atlas library:add \
  --type repository \
  --url https://github.com/org/repo \
  --display-name "baseline repo" \
  --format=json
```

Then check status:

```bash
atlas library:show --source-id src_... --format=json
atlas library:tree --source-id src_... --format=json
```

If indexing is still running, wait and call `library:show` again.

## Search before asking

```bash
atlas library:grep \
  --source-id src_... \
  --pattern "validation loss" \
  --format=json

atlas library:search \
  --query "where is the baseline loss computed?" \
  --format=json
```

Use grep when you know the string. Use search when you know the concept.

## Ask a grounded question

```bash
atlas library:ask \
  --query "Summarize the evidence for the baseline metric and cite the relevant files." \
  --format=json
```

For longer research work, use an async job:

```bash
atlas research:runs:create \
  --payload '{"query":"Map the strongest objections to this baseline."}' \
  --format=json

atlas research:runs:stream --job-id job_...
```

## Save the useful result

```bash
atlas nodes:create \
  --payload-json '{
    "local_temp_node_id": "tmp-source-audit",
    "parent_ids": ["nd_root"],
    "staged_payload": {
      "title": "baseline metric source audit",
      "content": "The baseline metric is computed in src/eval.py and logged in runs/baseline/metrics.json.",
      "summary": "Source-backed observation about baseline metric provenance.",
      "kind": "observation"
    }
  }' \
  --format=json
```

Do not save every search result. Save only the claims, observations, plans, or open questions that should survive beyond the search session.
