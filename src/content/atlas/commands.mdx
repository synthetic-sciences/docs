This page mirrors the shipped Atlas command registry. For live machine-readable metadata, run:

```bash
atlas help --format=json
atlas help nodes:create --format=json
```

The current CLI exposes `106` commands. Local commands run inside the CLI runtime; every other command maps to an Atlas `/api/v1` route.

## Command rename migration

Atlas 0.5.10 uses `nodes:*` for durable graph objects. Update scripts and prompts from old names like `entry:create`, `entry:save`, `draft:claim`, and `evidence:entry-files` to `nodes:create`, `nodes:save`, `draft:lock`, and `nodes:files`.

Older endpoint-shaped names such as `nodes:commit-new`, `graph:get`, `artifacts:upload`, and `sources:create` remain removed. REST routes behind the CLI did not change.

## How to read the reference

- Use `--format=json` for agent automation and scripts.
- Use `--out <path>` when exporting files or machine-readable payloads.
- Use `--yes` only for destructive commands after checking target IDs.
- Use `--idempotency-key <key>` for retries around writes.

## Global flags

| Flag | Use |
| --- | --- |
| `--format <json|tsv|csv|table|pretty>` | Select output format. Use `json` for agents. |
| `--env <profile>` | Select a stored Atlas CLI profile. |
| `--api-key <key>` | Override the API key for one invocation. Prefer environment variables in shared logs. |
| `--base-url <url>` | Override the API base URL for one invocation. |
| `--out <path>` | Write output to a file. Use this for binary exports and downloads. |
| `--yes` | Confirm destructive commands. |
| `--force` | Force a supported safety-gated operation. |
| `--idempotency-key <key>` | Provide a stable key for retried writes. |
| `--timeout <seconds>` | Override request timeout. |
| `--no-retry` | Disable transient retry behavior. |
| `--debug` | Print structured request debugging. |
