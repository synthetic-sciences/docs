Evidence records attach durable files to map nodes. Use them for run outputs, logs, PDFs, datasets, notes, screenshots, and other raw support.

Evidence is for raw support. Nodes are for interpretation. Keep that separation and future reviews become much easier.

## Commands

| Command | Use |
| --- | --- |
| `evidence:add` | One-shot prepare, file upload, and finalize flow. |
| `evidence:prepare` | Prepare one or more uploads when a custom upload flow is needed. |
| `evidence:finalize` | Finalize prepared upload batches. |
| `evidence:list` | List evidence attached to one node. |
| `evidence:show` | Inspect one evidence item. |
| `evidence:note` | Add, update, or clear an evidence note. |
| `evidence:delete` | Delete one evidence item. |
| `evidence:delete-bulk` | Delete multiple evidence items from one node. |
| `evidence:files` | List all files owned by the current user across nodes. |

## Upload

The high-level upload command prepares the upload, writes the file, and finalizes the record.

```bash
atlas evidence:add \
  --node-id nd_... \
  --items '[{"local_path":"./runs/baseline.json","artifact_type":"json","note":"baseline metrics"}]' \
  --format=json
```

The one-shot uploader accepts up to `50` files and `100 MB` per file. Supported artifact types include `text`, `table`, `json`, `image`, `banner`, `html`, `plotly_html`, `vega`, `checkpoint`, `binary`, and `diff_carousel`.

## Lower-level upload flow

Use these commands when you need custom upload control:

```bash
atlas evidence:prepare \
  --node-id nd_... \
  --items '[{"filename":"baseline.json","artifact_type":"json","media_type":"application/json"}]'

atlas evidence:finalize --node-id nd_... --batch-token batch_...
```

Use the lower-level flow when an external uploader needs the prepared upload target before finalization.

## Inspect files

```bash
atlas evidence:list --node-id nd_... --format=json
atlas evidence:show --node-id nd_... --artifact-id art_... --format=json
atlas evidence:files --format=json
```

## Notes

Evidence notes are lightweight metadata, not a replacement for node saves.

```bash
atlas evidence:note \
  --node-id nd_... \
  --artifact-id art_... \
  --note "contains baseline metrics and raw stderr" \
  --format=json
```

## Destructive actions

Use `--yes` only after checking the target IDs:

```bash
atlas evidence:delete --node-id nd_... --artifact-id art_... --yes
atlas evidence:delete-bulk --node-id nd_... --artifact-ids art_1,art_2 --yes
```
