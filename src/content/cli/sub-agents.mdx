The default `synsci` profile is a general coding agent. When a task needs a different personality, a different tool set, or a different routing policy, scaffold a sub-agent.

A **sub-agent** is an agent profile that the main agent can delegate to. From inside a session, you call it like a tool. From the CLI, you can also invoke it directly as the primary agent for a session.

## Scaffold a sub-agent

```bash
synsci agent create
```

`agent create` walks an interactive scaffold. The non-interactive form:

```bash
synsci agent create \
  --path .synsc/agents/reviewer.md \
  --description "Reviews diffs for race conditions and unsound concurrency" \
  --mode subagent \
  --tools "Read,Grep,Bash" \
  --model anthropic/claude-opus-4-7
```

Flags:

| Flag | Use |
| --- | --- |
| `--path <dir-or-file>` | Where to write the agent definition. Project-local under `.synsc/agents/` or user-global under `~/.config/synsc/agents/`. |
| `--description <text>` | What the agent is for. Surfaced in the picker and to the main agent when it decides whether to delegate. |
| `--mode <all\|primary\|subagent>` | `primary` means it can be used as the top-level agent for a session. `subagent` means only the main agent can call it. `all` allows both. |
| `--tools <tool[,tool...]>` | Restrict the tool set. Omit to inherit everything. |
| `--model <provider/model>` | Default model for this agent. Overrides the global default. |

The resulting file is a normal Markdown document with YAML frontmatter (the manifest) and a body (the system prompt). Edit it like any prose file.

## Invoke a sub-agent

From inside a session, the main agent will pick a sub-agent automatically when delegation makes sense. You can also force it by name:

```text
Use the reviewer sub-agent to look at the last commit and report any
race conditions.
```

To start a session with a sub-agent as the primary agent:

```bash
synsci --agent reviewer
synsci run --agent reviewer "Review the diff in src/auth"
```

`--agent <name>` matches against any agent file the CLI can resolve (project-local first, then user-global, then built-in). Only agents with `mode: primary` or `mode: all` can be invoked this way.

## A useful sub-agent shape

A sub-agent is most useful when it does one thing well. Common shapes:

- **Reviewer.** Read-only tool set (`Read`, `Grep`, `Bash` limited to read-only commands). System prompt focused on a specific class of bug or style rule. Lower-cost model.
- **Explorer.** Read-only with broad search. Used by the main agent for "where is X defined" questions without polluting the main session's context.
- **Skill specialist.** A profile that pairs a sub-agent with one or two skills (e.g. `vllm-serving` + `evaluation-harness`) so the agent always loads them.
- **Pipeline driver.** A profile with a narrow tool set and a system prompt that knows exactly one workflow (e.g. "set up a Modal sandbox, run the eval, tear down").

Keep system prompts short. Long system prompts compete with the user's prompt for attention; a tight description plus a clear tool set usually does more work than a multi-page instruction.

## Tool restrictions

`--tools "Read,Grep"` produces a read-only sub-agent. The main agent will refuse delegations that require a tool the sub-agent does not have, so a read-only reviewer cannot be tricked into editing files.

Common tool names you might restrict to:

| Tool | What it does |
| --- | --- |
| `Read` | Read files. |
| `Grep` | Search inside files. |
| `Glob` | Find files by name pattern. |
| `Bash` | Run shell commands. |
| `Edit` | Mutate files. |
| `Write` | Create or overwrite files. |
| `MCP` | Call tools exposed by configured MCP servers. |

The exact name set comes from your CLI version; run `synsci agent create --help` for the current list.

## Where agents live

```text
.synsc/agents/             # project-scoped agents
~/.config/synsc/agents/    # user-global agents
<bundled>                  # ships-with-CLI defaults
```

The CLI resolves in that order: a project-local `reviewer` beats a user-global `reviewer`, which beats a built-in `reviewer`.
