This is the wide-angle view of what ships in the `synsci` binary. Use it to find the right page when you know roughly what you want but not where it lives.

## Sessions

| Feature | Where |
| --- | --- |
| TUI agent | bare `synsci`, see [Sessions](/sessions) |
| One-shot prompt | `synsci run`, see [Sessions](/sessions) |
| Continue or resume | `synsci run -c`, `synsci run -s <id>`, see [Sessions](/sessions) |
| File and remote attachments | `-f`, `--attach`, see [First session](/first-session) |
| Generate (no TUI) | `synsci generate` |
| List sessions | `synsci session list` |
| Show session | `synsci session show <id>` |
| Export / import | `synsci export`, `synsci import` |

## Models and providers

| Feature | Where |
| --- | --- |
| Provider sign-in | `synsci auth`, see [Credentials](/credentials) |
| OpenAI Codex (Sign-in-with-ChatGPT) | `synsci auth` → OpenAI Codex, see [Codex](/codex) |
| List configured providers | `synsci models`, see [Models](/models) |
| Per-call model override | `--model <provider>/<id>`, see [Models](/models) |
| BYOK vs managed routing | dashboard + `synsci billing show`, see [Models](/models) |
| Credit and topup | `synsci billing show`, `synsci billing topup` |

## Skills

| Feature | Where |
| --- | --- |
| Built-in ML skill library | 82 skills × 20 categories, see [Skills](/skills) |
| Install a skill | `synsci skill add <url-or-name>` |
| List or inspect skills | `synsci skill list`, `synsci skill show` |
| Remove a skill | `synsci skill remove` |
| Pin skills to a project | `.synsc/skills.json`, see [Skills](/skills) |
| Claude Code marketplace | `ai-research-skills` bundle, see [Skills](/skills) |

## Sub-agents

| Feature | Where |
| --- | --- |
| Scaffold an agent profile | `synsci agent create`, see [Sub-agents](/sub-agents) |
| Tool restrictions | `--tools`, see [Sub-agents](/sub-agents) |
| Per-agent default model | `--model` on `agent create`, see [Sub-agents](/sub-agents) |
| Project-local vs user-global | `.synsc/agents/` vs `~/.config/synsc/agents/` |

## Servers and protocols

| Feature | Where |
| --- | --- |
| Local browser UI | `synsci web`, see [Web UI](/web-ui) |
| HTTP API server | `synsci serve`, see [Server mode](/server-mode) |
| Attach a TUI to a running server | `synsci tui attach`, `synsci tui thread` |
| Agent Client Protocol (stdio) | `synsci acp`, see [Server mode](/server-mode) |
| Model Context Protocol | `synsci mcp`, see [Server mode](/server-mode) |
| Server auth | `SYNSCI_SERVER_PASSWORD`, see [Security](/security) |

## Connect and accounts

| Feature | Where |
| --- | --- |
| Device-flow login | `synsci connect login`, see [Connect](/connect) |
| Account and credential status | `synsci connect status` |
| Refresh synced credentials | `synsci connect sync` |
| List devices on the account | `synsci connect devices` |
| Revoke a device locally | `synsci connect logout` |

## Runtime and diagnostics
