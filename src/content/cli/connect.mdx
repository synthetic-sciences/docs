`synsci connect` is the bridge between the local CLI and your dashboard account. It uses a device-code OAuth flow so the CLI never has to handle your password and can run on machines that do not have a browser.

The resulting credential is a `thk_<uuid>.<secret>` Bearer token with a one-year TTL. It is stored under `$XDG_CONFIG_HOME/synsc` (defaults to `~/.config/synsc`), scoped to the device that approved it, and revocable from the dashboard.

## The device flow

When you run `synsci connect login`, the CLI:

1. Calls the Synci backend for a device code and matching user code.
2. Prints the verification URL and the user code, then attempts to open your browser.
3. Polls the backend for completion, retrying on the cadence the server sets.
4. On approval, stores the Bearer token locally and runs `connect sync` to pull synced credentials.

The verification URL defaults to [app.syntheticsciences.ai/cli](https://app.syntheticsciences.ai/cli). Override with `SYNSC_AUTH_URL` if you are pointing at a staging or self-hosted dashboard.

```bash
synsci connect login
# Open: https://app.syntheticsciences.ai/cli
# Code: SQXR-7HVL
```

If the browser does not open (headless box, SSH session, no display), copy the URL into a browser on a machine you trust, sign in, and enter the user code.

## Subcommands

| Command | Use |
| --- | --- |
| `synsci connect login` | Run the device-flow login and store the Bearer token. |
| `synsci connect status` | Show the active email, token age, synced services, default model, and credit balance. |
| `synsci connect logout` | Invalidate the local token. The token is also revoked server-side. |
| `synsci connect sync` | Re-pull synced credentials from the dashboard without a new login. |
| `synsci connect devices` | List devices that hold an active Bearer token for your account. |

`connect status` is the right first stop when something feels off. It surfaces stale credentials, missing models, and routing issues in one place.

## Environment overrides

| Variable | Default | Purpose |
| --- | --- | --- |
| `SYNSC_AUTH_URL` | `https://app.syntheticsciences.ai/cli` | Verification URL the device flow opens in your browser. |
| `SYNSC_API_BASE` | `https://thesis-synsc.fly.dev` | Synci backend base URL. The CLI calls this for sync, billing, and device approval. |
| `XDG_CONFIG_HOME` | `~/.config` | Parent directory of `synsc/` where the Bearer token and synced credentials live. |

Set these in your shell rc when you want a non-default backend, a separate config path for a second account, or to point the CLI at a staging environment.

## Multiple accounts

Each device approval is independent, so you can keep one account on your laptop and a different one on a remote dev box without them stepping on each other. On a single machine, switch by changing `XDG_CONFIG_HOME` before running `synsci connect login` for the second account; the resulting `~/.config/synsc/` tree is fully scoped.
