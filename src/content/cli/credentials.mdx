The CLI handles two kinds of credentials. **Model provider credentials** route inference calls (Anthropic, OpenAI, OpenAI Codex, Google, OpenRouter, Together, Groq, Fireworks, and the managed Synthetic Sciences route). **Synced service credentials** are third-party keys you store in the dashboard once and pull into local sessions for compute, training, datasets, and observability (HF, W&B, Modal, Lambda, RunPod, Prime Intellect, TensorPool, Vast, Tinker, LangSmith, Pinecone).

You do not paste any of these into shell rc files. The dashboard holds the canonical copy and the CLI fetches it on demand.

## Model provider auth

```bash
synsci auth          # interactive provider picker
synsci models        # list everything configured
```

`auth` walks a menu of every supported provider. Each provider has its own flow: an OAuth handshake (for [OpenAI Codex](/codex)), an API key paste (Anthropic, OpenAI, Google, OpenRouter, Together, Groq, Fireworks), or the managed route (no input; just enabled per account). Once configured, models from that provider become available to the agent immediately and persist across sessions.

`models` prints what is currently configured, including the routing label the agent uses internally. The default model is set in your dashboard and can be overridden per call with `--model <provider>/<model>`.

To remove a provider, run `synsci auth` and pick "sign out" from the provider's submenu.

## Synced service credentials

Synced credentials come from the dashboard at [app.syntheticsciences.ai/cli](https://app.syntheticsciences.ai/cli). You paste them once in the dashboard's Credentials panel; the CLI pulls them in on `connect login` and refreshes them on `connect sync`.

| Service | Env name | Used for |
| --- | --- | --- |
| Hugging Face | `HF_TOKEN` | Private models, datasets, Spaces, gated repos. |
| Weights & Biases | `WANDB_API_KEY` | Training logs, sweeps, artifacts. |
| Modal | `MODAL_TOKEN_ID`, `MODAL_TOKEN_SECRET` | Serverless GPU jobs. |
| Lambda Cloud | `LAMBDA_API_KEY` | On-demand GPU instances. |
| RunPod | `RUNPOD_API_KEY` | Serverless and dedicated GPUs. |
| Prime Intellect | `PRIME_INTELLECT_API_KEY` | GPU compute marketplace. |
| TensorPool | `TENSORPOOL_API_KEY` | Spot GPU jobs. |
| Vast.ai | `VAST_API_KEY` | Spot GPU marketplace. |
| Tinker | `TINKER_API_KEY` | Hosted fine-tuning. |
| LangSmith | `LANGSMITH_API_KEY` | Agent traces and evaluations. |
| Pinecone | `PINECONE_API_KEY` | Vector search. |

These keys are passed to subprocesses through a curated allow-list (see [Security](/security)), and only when the agent runs a command that needs them. They never get written to disk on your machine and never appear in agent transcripts.

## Updating a credential

Change the value in the dashboard. Trigger a refresh locally with:

```bash
synsci connect sync
```

The next subprocess invocation will see the new value. There is no local cache to clear.

## BYOK vs managed routing

For model providers, the CLI supports both **BYOK** (you provide an API key, the call goes directly from your machine to the provider) and the **managed Synthetic Sciences route** (the call goes through `https://thesis-synsc.fly.dev` and bills against your account credit). `synsci models` shows which path is active for each provider; `synsci billing show` shows current credit balance and recent usage.

```bash
synsci billing show
synsci billing topup
```

## Local file layout

```text
~/.config/synsc/
  config.json        # account binding, default profile
  credentials.json   # synced service credentials (encrypted at rest)
  sessions/          # local session metadata
```

The Bearer token from `connect login` lives in `config.json`. Synced credentials live in `credentials.json`, encrypted at rest. Override the parent directory with `XDG_CONFIG_HOME`.
