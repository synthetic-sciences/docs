Synci ships through three install paths. They produce slightly different filesystem layouts but a compatible binary surface. Pick the one that fits how the rest of your environment is managed.

## npm

The launcher package on npm is [`synsci`](https://www.npmjs.com/package/synsci). It resolves to the heavier `@synsci/cli` package on first run and pins to it for subsequent invocations.

```bash
npm i -g synsci
synsci --version
```

Requirements: Node `18+`. The npm install puts the `synsci` binary on your PATH through your global Node bin directory.

If you would rather skip the launcher and install the implementation package directly:

```bash
npm i -g @synsci/cli
synsci --version
```

Both expose the same `synsci` entry point.

## Homebrew

```bash
brew install SyntheticSciences/tap/synsc
synsc --version
```

Homebrew installs the native binary as `synsc` (no trailing `i`). The commands and flags are identical to the npm install; only the entry point name differs.

## curl-bash installer

The fastest path on a fresh machine:

```bash
curl -fsSL https://syntheticsciences.ai/install | bash
synsc --version
```

The installer detects your OS and CPU architecture, downloads the matching release binary, places it at `~/.synsc/bin/synsc`, and adds that directory to your shell rc so subsequent shells pick it up automatically.

Supported targets: macOS `arm64` and `x64`, Linux `arm64` and `x64` (including `baseline` builds for non-AVX2 CPUs and `musl` builds for Alpine), and Windows `x64`.

Useful installer flags:

| Flag | Use |
| --- | --- |
| `-v, --version <version>` | Pin to a specific release instead of `latest`. |
| `-b, --binary <path>` | Install the binary to a different path. |
| `--no-modify-path` | Skip shell-rc editing. You will need to put the binary on PATH manually. |

## Verify the runtime

After any install path, confirm the binary resolves and the version reads back cleanly:

```bash
synsci --version
synsci --help
```

If `synsci` is not on PATH after install, open a new shell, or `source` the rc file that the installer edited. The npm install relies on your Node global bin directory; the curl-bash install relies on `~/.synsc/bin`.

## Upgrade

```bash
synsci upgrade
```

`upgrade` detects how you installed (npm, Homebrew, or curl-bash binary) and runs the matching update flow. Pin a version with `synsci upgrade --version 1.1.122` to roll back or stay on a tested release.

## Uninstall

```bash
synsci uninstall
```

This removes the binary and clears local config under `~/.config/synsc`. Your dashboard account, billing balance, and synced credentials are unaffected; reinstall any time with the same login.
