If you want a coding agent (Claude Code, Codex CLI, Cursor, or a sub-agent inside Synci itself) to set up Synci on a fresh machine, paste the prompt below. It is also a useful onboarding checklist for a human teammate.

## The prompt

```text
Set up the Synci CLI on this machine end-to-end.

1. Detect the install path that fits this environment:
   - If `npm` is available and Node 18+ is installed, run `npm i -g synsci`.
   - Else if `brew` is available, run `brew install SyntheticSciences/tap/synsc`.
   - Else, run `curl -fsSL https://syntheticsciences.ai/install | bash`
     and confirm `~/.synsc/bin` is on PATH (open a new shell if it isn't).

2. Verify the binary:
   - `synsci --version` (or `synsc --version` for the brew/curl install)
   - The command must exit zero and print a version >= 1.1.0.

3. Run the device-flow login:
   - `synsci connect login`
   - Print the verification URL and user code to the human running the prompt.
   - Wait for the polling to complete. Do not retry the login itself.

4. Verify the connection:
   - `synsci connect status`
   - The output must show an authenticated email, an active Bearer token,
     and at least one configured model.

5. If no model is configured, prompt the human to run `synsci auth` and pick
   either OpenAI Codex (sign-in with ChatGPT), a BYOK provider, or the managed
   Synthetic Sciences route. Do not paste API keys for them.

6. Smoke-test the agent loop:
   - `synsci run "Reply with the literal text OK and nothing else"`
   - The command must exit zero and print "OK".

Report back the final state: install path used, binary version, connected
account email, configured providers, and any step that needed human input.
Do not write API keys to any file, do not commit anything, and do not run
destructive commands.
```

## Why this works

The prompt enumerates the three install paths, gives the agent fall-through logic instead of guessing, and pins the verification steps that actually catch a broken install (`connect status` for the bridge, a `run` smoke test for the agent loop). The "do not paste API keys" line is load-bearing — agents are otherwise eager to fabricate credentials.

If the agent reports a failure at step 4 ("status shows no token"), the usual fix is a re-run of `connect login` with a working browser. If it fails at step 6 ("no model"), the human needs to complete `synsci auth` interactively; agents cannot drive the OAuth pop-up.

## A scoped variant

For an agent that only needs to verify an existing install (not set one up), use:

```text
Verify that the Synci CLI on this machine is healthy.

Run:
- `synsci --version`
- `synsci connect status`
- `synsci models`
- `synsci run "Reply with the literal text OK"`

Report version, account, configured providers, and the result of the smoke
test. If any step fails, report which one and the exact error message.
Do not attempt to repair anything.
```
