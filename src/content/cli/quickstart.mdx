This walks you from a fresh shell to a working session with a signed-in account, a model provider, and your first prompt. It assumes Node `18+` is installed.

<Steps>
  <Step title="Install the CLI">
    ```bash
    npm i -g synsci
    synsci --version
    ```

    See [Installation](/installation) for Homebrew and curl-bash alternatives.
  </Step>

  <Step title="Sign in to your account">
    ```bash
    synsci connect login
    synsci connect status
    ```

    `connect login` runs a device-code OAuth flow: it prints a verification URL and a one-time code, opens your browser to [app.syntheticsciences.ai/cli](https://app.syntheticsciences.ai/cli), and polls until you approve the device. The resulting `thk_*` Bearer token is stored under `~/.config/synsc`. `connect status` confirms the active email, synced services, and credit balance.
  </Step>

  <Step title="Add a model provider">
    The agent needs at least one model route. The simplest options:

    ```bash
    synsci auth          # interactive provider picker (BYOK or OAuth)
    synsci models        # list everything that's configured
    ```

    You can sign in to OpenAI Codex (Sign-in-with-ChatGPT), paste an Anthropic or OpenAI API key, or rely on the managed Synthetic Sciences route if you have credit. See [Codex](/codex) for the GPT-5 Codex flow and [Credentials](/credentials) for dashboard-managed keys.
  </Step>

  <Step title="Send your first prompt">
    The bare command drops you into the TUI agent:

    ```bash
    synsci
    ```

    For a one-shot prompt that exits when the agent is done, use `run`:

    ```bash
    synsci run "Refactor utils/log.ts to use structured logging"
    ```

    For a local browser UI on `http://localhost:4444`:

    ```bash
    synsci web
    ```

    <Warning>
      The agent has the same filesystem and shell access you do. Run it in a project directory you trust, or run inside Docker if you want isolation.
    </Warning>
  </Step>
</Steps>

## What success looks like

`synsci connect status` should show your email, an active Bearer token, at least one provider under "models", and a non-zero credit balance if you intend to use the managed route. A bare `synsci` should open a TUI; `synsci run "echo hello"` should print a short reply and exit.

If login succeeds but commands fail, check `synsci connect status` before changing config. It is the fastest way to separate a local config issue from a provider routing issue.
