---
title: "synci codex"
description: "Sign in to OpenAI Codex from the terminal so the agent can invoke codex and codex exec on demand."
---

`synci codex` manages the OpenAI Codex token alongside your Synci Bearer token. After the browser flow completes, the Codex token sits in the OS keychain next to your Synci Bearer token, and is injected into Codex-aware subprocesses (`codex`, `codex exec`, etc.) on demand.

| Subcommand | Purpose |
|---|---|
| `synci codex login` | Open the Codex sign-in flow in your browser and store the returned token. |
| `synci codex status` | Show the active Codex account, token expiry, and which workspaces are reachable. |
| `synci codex logout` | Revoke and remove the local Codex token. |

## Logging in

<Steps>
  <Step title="Start the flow">
    ```bash
    synci codex login
    ```

    The CLI opens a browser tab pointing at the Codex sign-in screen. Approve the requested scopes (workspaces you want the CLI to be able to use).
  </Step>

  <Step title="Pin a specific workspace (optional)">
    ```bash
    synci codex login --workspace my-org
    ```

    Binds the token to that workspace slug. Without `--workspace`, the token is bound to your personal workspace.
  </Step>

  <Step title="Headless boxes">
    ```bash
    synci codex login --no-browser
    ```

    Prints a device-code URL instead of trying to open a browser. Open the URL on any device that has a browser, sign in, and the CLI picks up the token automatically.
  </Step>
</Steps>

## Checking the status

```bash
synci codex status
```

Reports the active Codex account, token expiry, and the workspaces the token can reach. The agent picks the right workspace based on the active mode and your dashboard defaults.

## Flags

- `--workspace <slug>` to bind the token to a specific Codex workspace.
- `--no-browser` to print a device-code URL instead of opening a browser.

<Warning>
  Codex tokens are scoped per workspace. Re-run `synci codex login --workspace <slug>` to add another workspace; tokens for different workspaces coexist in the keychain.
</Warning>
