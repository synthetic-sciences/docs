Synci integrates with OpenAI Codex through the same sign-in-with-ChatGPT flow you use on chatgpt.com. After the OAuth handshake completes, your Codex session is stored alongside the other provider credentials and routed automatically when you select an `openai-codex` model.

This is the path to use when you already pay for ChatGPT or Codex and would rather not generate a separate OpenAI API key.

## How it works

The CLI runs an OAuth Authorization-Code flow with PKCE against [auth.openai.com](https://auth.openai.com). The browser callback is captured by a short-lived local listener on port `1455`, the tokens are exchanged for a session, and the session is registered with the Synthetic Sciences backend so the agent can refresh it through your dashboard account. Once registered, the provider appears in `synsci models` as **"Signed in with Codex"**.

The system prompt the agent uses for this provider opens with: "You are a coding agent powered by OpenAI GPT-5 Codex models, running inside Synthetic Sciences CLI." You can mix Codex with any other configured provider in the same session; the agent picks per call based on routing rules.

## Sign in

```bash
synsci auth
# pick "OpenAI Codex" from the provider menu
```

The CLI opens your browser to the OpenAI sign-in page. After you approve the requested scopes, the browser is redirected back to `http://localhost:1455/...`, the local listener captures the code, and the CLI prints the registered account. If a browser does not open automatically, the CLI prints the verification URL so you can paste it manually on a device that has a browser.

Confirm the result with:

```bash
synsci models
```

You should see `openai-codex` listed with **"Signed in with Codex"** and the email tied to your ChatGPT account.

## Use Codex in a session

Once signed in, route to Codex by selecting an `openai-codex` model:

```bash
synsci run --model openai-codex/gpt-5-codex "Refactor utils/log.ts to use structured logging"
```

You can also set Codex as the default model in your dashboard, which makes a bare `synsci` and `synsci run` route to it without an explicit `--model` flag.

## Sign out

Removing the Codex session is done through the same provider menu:

```bash
synsci auth
# pick "OpenAI Codex" → sign out
```

This revokes the local session and removes the registration on the dashboard side. You can re-run `synsci auth` any time to sign back in.

## Headless boxes

The local callback listener needs port `1455` to be free on the machine you are signing in from, which can be a problem on a headless SSH box. The standard pattern is to sign in once on a machine that has a browser (the same `~/.config/synsc` directory will hold the session) and then move or re-create the config on the headless box, or to use a different provider (an `OPENAI_API_KEY` BYOK route or the managed Synthetic Sciences route) on headless machines.
