This page mirrors the shipped command registry. For the live machine-readable surface, run:

```bash
synsci --help
synsci connect --help
synsci skill --help
```

The bare `synsci` (no arguments) opens the TUI agent in your working directory with the default profile and the default model. Every other top-level command is grouped below by area.

## Global flags

| Flag | Use |
| --- | --- |
| `-v, --version` | Print the installed CLI version. |
| `-h, --help` | Print help for the command or subcommand. |
| `--print-logs` | Stream agent diagnostics to stderr while the command runs. |
| `--log-level <DEBUG\|INFO\|WARN\|ERROR>` | Override the log threshold for this invocation. |

## Sessions

| Command | Use |
| --- | --- |
| `synsci` | Start the TUI agent. Default model, default profile, current working directory. |
| `synsci run [message..]` | Run a one-shot prompt. Streams the response, then exits. |
| `synsci generate` | Generate code or files via the agent without entering the TUI. |
| `synsci session` | Inspect and manage past sessions stored locally and on the dashboard. |
| `synsci tui attach` | Attach to a running agent process (e.g. one started via `synsci serve`). |
| `synsci tui thread` | Open a specific thread inside an attached agent. |

`run` flags worth knowing:

| Flag | Use |
| --- | --- |
| `-c, --continue` | Continue the most recent session for this directory. |
| `-s, --session <id>` | Resume a specific session by id. |
| `-m, --model <provider/model>` | Override the default model for this run. |
| `--agent <name>` | Use a specific agent profile (built-in or one you created). |
| `--format <default\|json>` | Switch to JSON output for scripting. |
| `-f, --file <path>` | Send a file as part of the prompt. |
| `--title <text>` | Set the session title. |
| `--attach <url>` | Attach a remote artifact (URL) to the prompt. |

```bash
synsci run "Refactor utils/log.ts to use structured logging"
synsci run -c "Now add tests"
synsci run --model openai-codex/gpt-5-codex "Plan a migration off the old auth middleware"
synsci run --format json "Summarize the architecture of src/" > summary.json
```

## Connect

| Command | Use |
| --- | --- |
| `synsci connect login` | Device-flow OAuth against the Synthetic Sciences dashboard. |
| `synsci connect status` | Active email, token age, default model, synced services, credit balance. |
| `synsci connect logout` | Revoke the local Bearer token. |
| `synsci connect sync` | Re-pull synced credentials from the dashboard. |
| `synsci connect devices` | List devices that hold an active Bearer token for your account. |

See [Connect](/connect) for the full flow.

## Providers and routing

| Command | Use |
| --- | --- |
| `synsci auth` | Interactive provider sign-in: BYOK API keys (Anthropic, OpenAI, Google, Together, Groq, Fireworks, OpenRouter) and OAuth (OpenAI Codex). |
| `synsci models` | List configured providers and the models they expose. |
| `synsci billing show` | Account credit balance and recent usage on the managed route. |
| `synsci billing topup` | Open the dashboard to top up account credit. |

```bash
synsci auth
synsci models
synsci billing show
```

See [Codex](/codex) for the OpenAI Codex sign-in-with-ChatGPT path and [Credentials](/credentials) for the full provider list.

## Agent profiles

| Command | Use |
| --- | --- |
| `synsci agent create` | Scaffold a new sub-agent (system prompt + tool set + routing policy). |

`agent create` flags:

| Flag | Use |
| --- | --- |
| `--path <dir>` | Where to write the agent definition. |
| `--description <text>` | Description shown in `synsci models` and the agent picker. |
| `--mode <all\|primary\|subagent>` | How the agent can be invoked. |
| `--tools <tool[,tool...]>` | Restrict the tool set. |
| `--model <provider/model>` | Default model for this agent. |

## Servers and protocols

| Command | Use |
| --- | --- |
| `synsci serve` | Run the CLI as an HTTP API server. |
| `synsci web` | Boot the local server and open the browser UI on `http://localhost:4444`. |
| `synsci acp` | Speak the Agent Client Protocol over stdio for host integrations. |
| `synsci mcp` | Manage MCP server connections. |

```bash
synsci serve --port 4444
synsci web
synsci web --port 5555
```

`serve` requires `SYNSCI_SERVER_PASSWORD` to enable HTTP Basic Auth before you expose the port beyond `127.0.0.1`. See [Security](/security).

## Portability

| Command | Use |
| --- | --- |
| `synsci export` | Export a session as a JSON document. |
| `synsci import` | Import a session JSON. |

## Diagnostics and lifecycle

| Command | Use |
| --- | --- |
| `synsci stats` | Local usage stats. |
| `synsci debug` | Print structured diagnostics for support tickets. |
| `synsci upgrade` | Update to the latest published version. Detects npm, Homebrew, or curl-bash install. |
| `synsci uninstall` | Remove the binary and clear local config. |
| `synsci completion` | Shell completion script for `bash`, `zsh`, or `fish`. |

```bash
synsci upgrade
synsci upgrade --version 1.1.122
synsci completion zsh > ~/.zsh/completions/_synsci
```
