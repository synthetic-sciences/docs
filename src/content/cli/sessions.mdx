A **session** is a single agent conversation: a thread of prompts and replies, the tool calls the agent made, and the working directory those tool calls ran against. Sessions are local-first — they live on your machine first and sync to the dashboard when you are connected.

The CLI exposes three ways to start a session and a handful of ways to revisit it.

## Start a session

| Entrypoint | Use |
| --- | --- |
| `synsci` | Open the TUI agent. Default profile, default model, current directory. |
| `synsci run [message..]` | Send one prompt, let the agent finish, and exit. Good for shell pipelines, CI, and git hooks. |
| `synsci web` | Boot the local server and open the browser UI on `http://localhost:4444`. Good for longer conversations. |
| `synsci generate` | Generate code or files via the agent without entering the TUI. |

The bare `synsci` and `synsci web` are interactive; `synsci run` is the one to script. All four produce a normal session that you can resume later.

## Continue or resume

| Command | Use |
| --- | --- |
| `synsci run -c [message..]` | Continue the most recent session in the current directory. |
| `synsci run -s <session-id> [message..]` | Resume a specific session by id. |
| `synsci session list` | List recent sessions for the active account. |
| `synsci session show <session-id>` | Print the prompts, assistant messages, and tool calls of one session. |

```bash
synsci session list
# SES_4XJ3K  utils/log.ts refactor      2 hours ago
# SES_2NQ9M  auth middleware rewrite    yesterday

synsci run -s SES_4XJ3K "Now add tests for the refactored module"
```

`-c, --continue` is the same as `-s` against the most recent id in the current directory. It is the most common form.

## Attach to a running agent

When `synsci serve` is running (locally or on a remote box you can reach), you can attach a TUI to it instead of starting a fresh session:

```bash
synsci tui attach           # attach to the default local server
synsci tui thread <id>      # attach and open a specific thread
```

This is how you drive a long-running agent from a different terminal, or share a session with a teammate who is on the same dev box. See [Server mode](/server-mode) for how to expose the server safely.

## Session output

`synsci run` defaults to a streaming, human-readable transcript. For scripting:

```bash
synsci run --format json "Summarize the architecture of src/"
```

JSON output emits one envelope per event (prompt, reply, tool call, tool result, completion). Parse it line-by-line, or capture the final envelope for a structured result.

```bash
synsci run --format json --title "src arch" "Summarize..." > arch.jsonl
```

`--title <text>` sets the title surfaced in `synsci session list` and on the dashboard.

## Export and import

```bash
synsci export <session-id> --out session.json
synsci import session.json
```

`export` writes a portable JSON document with the full session: prompts, replies, tool calls, results, and metadata. `import` accepts that document on another machine. Useful for sharing reproductions, attaching a session to an issue, or backing up before a destructive experiment.

## File and remote attachments

For one-shot prompts that need a file or URL:

```bash
synsci run -f data/sample.cif "Set up a DFT optimization for this file"
synsci run --attach https://arxiv.org/pdf/2305.13245 "Summarize the architecture in this paper"
```

`-f, --file <path>` reads the file from disk into the prompt. `--attach <url>` fetches a remote artifact at runtime. Multiple `-f` flags stack; multiple `--attach` flags stack.

## Session vs profile vs skill

Three concepts often confused on first read:

- A **session** is one conversation. It has an id, a transcript, and a working directory.
- An **agent profile** is the system prompt and tool set the agent uses inside that conversation. See [Sub-agents](/sub-agents).
- A **skill** is a portable instruction bundle layered on top of a profile. See [Research skills](/skills).

The same profile and the same set of skills can drive thousands of sessions; sessions are the cheap and disposable unit.
