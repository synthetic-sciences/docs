Research is reorganizing around delegation. Autoresearch loops, multi-agent harnesses, and AI scientists now run experiments in the background, and the volume of work a single researcher sustains keeps rising. **Atlas** is the infrastructure layer for that shift: a suite of tools by Synthetic Sciences that give human-agent research a durable, shared substrate.

Atlas is not one app. It is a set of products that work together so an investigation survives more than one chat, notebook, or agent run.

## How the pieces fit

One CLI, `@synsci/atlas`, drives the whole suite. It is a direct HTTP client for the Atlas API at `https://app.syntheticsciences.ai/api/v1`; there is no separate transport to configure.

```text
agent or human
  -> atlas <command>
  -> local profile + thk_* API key
  -> https://app.syntheticsciences.ai/api/v1
  -> graph, library, evidence, or integration service
  -> JSON (piped) or a pretty terminal view
```

Atlas is direct REST plus CLI. There is no Atlas MCP server. The CLI also bundles nine skills that teach any coding agent how to drive Atlas safely; `atlas install` drops them into the agents on your machine.

## The vocabulary

The product vocabulary is intentionally small and shared across the suite:

- A **node** is a durable research object: a question, hypothesis, experiment plan, recorded run, decision, claim, or source-grounded observation.
- A **link** records dependency or lineage between nodes.
- **Evidence** attaches files, metrics, logs, or external JSON to the node it supports.
- A **library source** is indexed external material (a repo, doc site, paper, dataset, or local folder) the graph can search and cite.
- A **project** is the root node for one folder or repo. It is where the research loop runs.

## Minimum working setup

```bash
npm i -g @synsci/atlas@latest
atlas login
atlas doctor --format=json
```

`atlas login` opens a browser approval flow and stores an API key locally. `atlas doctor` confirms auth, backend reachability, and the nine bundled skills. From there, jump to [Atlas Graphs](/graphs/index) to run the research loop or [Atlas Library](/library/index) to index sources.
