The agent handles two kinds of credentials. **Model provider credentials** route inference (see [Models](/agent-cli/models)). **Synced service credentials** are third-party keys you store once on your account and pull into local sessions for compute, training, datasets, and observability.

You do not paste these into shell rc files. Your account holds the canonical copy and the CLI fetches it on demand.

## Synced service credentials

| Service | Env name | Used for |
| --- | --- | --- |
| Hugging Face | `HF_TOKEN` | Private models, datasets, gated repos. |
| Weights & Biases | `WANDB_API_KEY` | Training logs, sweeps, artifacts. |
| Modal | `MODAL_TOKEN_ID`, `MODAL_TOKEN_SECRET` | Serverless GPU jobs. |
| Lambda Cloud | `LAMBDA_API_KEY` | On-demand GPU instances. |
| RunPod | `RUNPOD_API_KEY` | Serverless and dedicated GPUs. |
| Prime Intellect | `PRIME_INTELLECT_API_KEY` | GPU compute marketplace. |
| TensorPool | `TENSORPOOL_API_KEY` | Spot GPU jobs. |
| Vast.ai | `VAST_API_KEY` | Spot GPU marketplace. |
| Tinker | `TINKER_API_KEY` | Hosted fine-tuning. |
| LangSmith | `LANGSMITH_API_KEY` | Agent traces and evaluations. |
| Pinecone | `PINECONE_API_KEY` | Vector search. |

These are passed to subprocesses through a curated allow-list (see [Security](/agent-cli/security)), and only when the agent runs a command that needs them. They are never written to disk on your machine and never appear in transcripts.

## Update a credential

Change the value in the app, then refresh locally:

```bash
synsci connect sync
```

The next subprocess sees the new value. There is no local cache to clear.

## Local file layout

```text
~/.config/synsc/
  config.json        # account binding, default profile
  credentials.json   # synced service credentials (encrypted at rest)
  sessions/          # local session metadata
```

Override the parent directory with `XDG_CONFIG_HOME`.
