A **model** is one inference endpoint the agent routes through; a **provider** is a vendor that exposes models. The agent routes locally — each call goes from your machine either directly to the provider (BYOK) or through the managed Synthetic Sciences route. There is no extra gateway in the middle when you BYOK.

## Supported providers

| Provider | Auth | Notes |
| --- | --- | --- |
| Anthropic | BYOK (API key) | Claude family. |
| OpenAI | BYOK (API key) | GPT family. |
| OpenAI Codex | OAuth (sign in with ChatGPT) | GPT-5.2 Codex. |
| Google | BYOK (API key) | Gemini family. |
| OpenRouter | BYOK (API key) | Aggregator for many models. |
| Together | BYOK (API key) | Hosted open-weights models. |
| Groq | BYOK (API key) | Fast inference for open-weights models. |
| Fireworks | BYOK (API key) | Hosted open-weights models. |
| Synthetic Sciences | Managed (account credit) | The default route when you have credit. No external key. |

## Configure a provider

```bash
synsci auth      # interactive picker
synsci models    # list everything configured
```

`auth` walks a menu: paste an API key, run the OAuth handshake for Codex, or enable the managed route. Credentials are stored encrypted at rest under `~/.config/synsc/`. To remove a provider, run `synsci auth` and pick "sign out".

## Pick a model per call

```bash
synsci run --model anthropic/claude-sonnet-4-6 "Review this diff for race conditions"
synsci run --model openai/gpt-5.2-codex "Plan a migration off the old auth middleware"
synsci run --model synsci/glm-4.7 "..."   # a managed-route model, no external key
```

`--model <provider>/<model-id>` overrides the default for one call. `synsci models` prints the full set per provider; the catalogue is models.dev-backed, so it tracks whatever models each provider currently exposes.

## BYOK vs the managed route

| Path | What runs where | Billing |
| --- | --- | --- |
| BYOK | Your machine calls the provider directly. | Your provider account. |
| Managed | Your machine calls the Synthetic Sciences route, which calls the provider. | Account credit. |

The managed route is the simplest start: one bill, no key juggling. BYOK is right when you already have a provider account or want exact provider invoices. You can mix routes in one install; the agent picks based on which provider the active model belongs to.

## Billing

```bash
synsci billing show     # account credit and recent managed-route usage
synsci billing topup    # open the app to add credit
```

BYOK calls do not appear here. Account credit is the same balance shown under [Billing & credits](/getting-started/billing).

## Subprocess key hygiene

The agent strips shared model keys (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, `GEMINI_API_KEY`) from any subprocess it spawns. It talks to providers itself; subprocesses do not need these keys, and stripping them prevents accidental exfiltration. See [Security](/agent-cli/security).
