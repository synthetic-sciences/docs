A **skill** is a portable instruction bundle the agent loads into the active session to prime it for a domain. Skills are pluggable knowledge that travels with a session and changes how the agent approaches a task.

<Warning>
These are the **agent's** research skills, distinct from the ten skills bundled in the Atlas CLI (`atlas`, `atlas-loop`, ...). For those, see [Graph skills](/graphs/skills) and the [CLI overview](/getting-started/cli-overview).
</Warning>

The agent ships an ML research-skills library covering the surface most teams hit: model architecture, tokenization, fine-tuning, RLHF and post-training, mechanistic interpretability, distributed training, inference serving, evaluation, MLOps, multimodal, RAG, and agents.

## Skill commands

| Command | Use |
| --- | --- |
| `synsci skill list` | Show installed skills, built-in and user-added. |
| `synsci skill show <name>` | Print a skill's metadata and entrypoint instructions. |
| `synsci skill add <url-or-name>` | Install from a URL (Git or HTTPS bundle) or by name. |
| `synsci skill remove <name>` | Remove an installed skill. |
| `synsci skill set-entries <name>` | Update which tools a skill injects and when it activates. |

## Built-in categories

| Category | Skills know about |
| --- | --- |
| Model architecture | Transformers, MoE, state-space models, attention variants, GQA, MLA. |
| Tokenization | BPE, Unigram, SentencePiece, vocabulary design. |
| Fine-tuning | Full FT, LoRA, QLoRA, adapter stacking. |
| Post-training / RLHF | DPO, GRPO, PPO, reward-model design. |
| Mechanistic interpretability | Probing, SAEs, activation patching, circuits. |
| Distributed training | FSDP, ZeRO, tensor/pipeline parallel, expert parallelism. |
| Inference serving | KV-cache management, speculative decoding, batching. |
| Evaluation | Harnesses, leaderboards, regression suites. |
| MLOps | Run tracking, artifact stores, deploy pipelines. |
| RAG | Chunking, embeddings, rerankers, hybrid retrieval. |
| Multimodal | Vision-language, audio, video. |
| GPU systems | CUDA kernels, Triton, profiling, NCCL. |

Run `synsci skill list` to see every skill on your install. Skills are versioned, so a project can pin a revision.

## Add a third-party skill

```bash
synsci skill add https://github.com/example-org/my-skill
synsci skill list
```

The CLI fetches the bundle, validates its manifest, and registers it under the declared name. It is then available in any future session.

## How a skill activates

A skill ships an entry manifest: a name, a description, an activation rule (always-on, project-scoped, or invoked by name), the tools it wants, and a system-prompt fragment. At session start the agent loads every always-on and project-scoped skill matching the working directory, plus any you invoke by name. Activation is additive — loading one skill does not unload another.

## Pin skills to a project

Create `.synsc/skills.json` in your project root:

```json
{
  "skills": [
    { "name": "vllm-serving", "version": "1.x" },
    { "name": "evaluation-harness", "version": "1.x" }
  ]
}
```

Every session started in that directory loads exactly those skills, so teammates and CI get the same primed agent.
