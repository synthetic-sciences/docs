`synsci web` is the same agent as the TUI, exposed through a local browser interface. The TUI is faster for tight loops; the web UI is better for longer conversations, multiple threads at once, and reviewing diffs and tool calls visually.

The server is local: it binds `127.0.0.1` by default, holds the agent in its own process, and serves the UI from the same port. Closing the browser does not kill the agent — reattach any time.

## Start the server

```bash
synsci web
```

By default it binds `127.0.0.1`, opens the system browser, and prints the URL (for example `http://localhost:4096`). It always binds localhost — the deprecated `server.hostname` / `server.mdns` config keys are ignored.

| Flag | Use |
| --- | --- |
| `--port <n>` | Bind a specific local port instead of the default. |
| `--cors <origin>` | Allow an extra CORS origin (repeatable). |

```bash
synsci web --port 5555
```

Choose the agent and model from inside the UI (the agent and model switcher), not from `synsci web` flags.

## macOS Full Disk Access

On macOS, the first `synsci web` invocation probes for Full Disk Access so the agent can read files outside the sandboxed Application Support tree (your home directory, project sources, dotfiles). If the probe fails, the UI shows a banner with instructions to grant access in System Settings → Privacy & Security; restart the server after granting.

## What the UI gives you

- **Threads.** Each conversation lives in its own thread; switch without losing context.
- **Tool-call viewer.** Every file read, diff, and shell command renders inline with its result.
- **Skill picker.** Toggle research skills mid-conversation; activation is additive.
- **Agent and model switcher.** Change the active agent or model per thread.
- **Attach pane.** Drag files or paste URLs onto the next prompt.

## Reuse a running server

A second `synsci web` reuses the running server instead of starting a new one. Attach a TUI from another terminal with the server URL:

```bash
synsci attach http://localhost:4096
synsci attach http://localhost:4096 -s <session-id>
```

The web UI is intended for local `127.0.0.1` use. Run it on a machine you control. To run a server without opening a browser, use `synsci serve` (secure it with `SYNSCI_SERVER_PASSWORD`).
