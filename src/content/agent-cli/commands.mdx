The SynSci agent CLI command surface. For the live help, run `synsci --help` and `synsci <command> --help`.

The bare `synsci` (no arguments) opens the TUI agent in your working directory with the default profile and model. Everything else is grouped below.

## Global flags

| Flag | Use |
| --- | --- |
| `-v, --version` | Print the installed version. |
| `-h, --help` | Print help for a command. |
| `--print-logs` | Stream agent diagnostics to stderr. |
| `--log-level <DEBUG\|INFO\|WARN\|ERROR>` | Override the log threshold. |

## Sessions

| Command | Use |
| --- | --- |
| `synsci` | Start the TUI agent. |
| `synsci run [message..]` | One-shot prompt; streams, then exits. |
| `synsci session list` | List recent sessions (`-n`, `--format`). |
| `synsci attach <url>` | Attach a TUI to a running server (e.g. one started by `synsci web` or `synsci serve`). |

`run` flags: `-c/--continue`, `-s/--session <id>`, `-m/--model <provider/model>`, `--agent <name>`, `--format <default|json>`, `-f/--file <path>`, `--attach <url>`, `--title <text>`.

```bash
synsci run "Refactor utils/log.ts to use structured logging"
synsci run -c "Now add tests"
synsci run --format json "Summarize the architecture of src/" > summary.json
```

## Account

| Command | Use |
| --- | --- |
| `synsci connect login` | Connect the CLI to your Synthetic Sciences / Atlas account (browser flow, or `--key thk_â€¦`). |
| `synsci connect status` | Show the active Atlas account, default model, synced services, and credit balance. |
| `synsci connect sync` | Re-pull synced credentials. |
| `synsci connect devices` | List authorized devices. |
| `synsci connect logout` | Disconnect this device. |

## Providers and routing

| Command | Use |
| --- | --- |
| `synsci auth` | Interactive provider sign-in (BYOK keys; OAuth for OpenAI Codex). |
| `synsci models` | List configured providers and their models. |
| `synsci billing show` | Account credit and recent managed-route usage. |
| `synsci billing topup` | Open the app to add credit. |

## Agent profiles

| Command | Use |
| --- | --- |
| `synsci agent create` | Scaffold a sub-agent (system prompt + tool set + routing policy). |

Flags: `--path`, `--description`, `--mode <all\|primary\|subagent>`, `--tools`, `--model`. See [Sub-agents](/agent-cli/sub-agents).

## Skills

| Command | Use |
| --- | --- |
| `synsci skill list` | Show installed skills (`--all` for the bundled set). |
| `synsci skill show <name>` | Print a skill's metadata and entrypoints. |
| `synsci skill add <url-or-name>` | Install a skill (URL or `gh:owner/repo`). |
| `synsci skill new` / `edit` / `validate` | Author, edit, and check user skills. |
| `synsci skill remove <name>` | Uninstall a skill. |
| `synsci skill set-entries <name>` | Update which skills surface in the `/` picker. |

## Local server and protocols

| Command | Use |
| --- | --- |
| `synsci web` | Boot the local server and open the browser UI (prints the URL, e.g. `http://localhost:4096`). |
| `synsci serve` | Boot a headless local server (no browser); secure with `SYNSCI_SERVER_PASSWORD`. |
| `synsci acp` | Start an Agent Client Protocol server for editors like Zed. |
| `synsci mcp` | Manage MCP servers (`list`, `add`, `remove`, `auth`). |

`web` and `serve` take `--port` and `--cors`. See [Web UI](/agent-cli/web-ui).

## GitHub and PRs

| Command | Use |
| --- | --- |
| `synsci github` | GitHub agent for CI/Actions (`install`, `run`). |
| `synsci pr <number>` | Check out a PR branch, then launch the agent on it. |

## Project and lifecycle

| Command | Use |
| --- | --- |
| `synsci project` | Pin the Atlas project root for this folder (`merge`). |
| `synsci export` / `synsci import` | Export or import a session as JSON. |
| `synsci generate` | Emit the OpenAPI spec for the local server to stdout. |
| `synsci stats` | Local token-usage and cost stats. |
| `synsci debug` | Structured diagnostics for support (`debug paths` shows data/config dirs). |
| `synsci upgrade` | Update to the latest version. |
| `synsci uninstall` | Remove the binary and local config. |
| `synsci completion` | Shell completion for bash, zsh, or fish. |

```bash
synsci upgrade
synsci completion zsh > ~/.zsh/completions/_synsci
```
