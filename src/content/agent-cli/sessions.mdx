A **session** is a single agent conversation: a thread of prompts and replies, the tool calls the agent made, and the working directory those calls ran against. Sessions are local-first and sync to your Atlas account when you are connected.

## Start a session

| Entrypoint | Use |
| --- | --- |
| `synsci` | Open the TUI agent. Default profile, default model, current directory. |
| `synsci run [message..]` | Send one prompt, let the agent finish, and exit. Good for pipelines, CI, and git hooks. |
| `synsci web` | Boot the local server and open the browser UI on `127.0.0.1:4444`. Good for longer conversations. |
| `synsci generate` | Generate code or files without entering the TUI. |

The bare `synsci` and `synsci web` are interactive; `synsci run` is the one to script. All four produce a session you can resume.

## Continue or resume

```bash
synsci session list
synsci run -c "Now add tests for the refactored module"   # latest session in this dir
synsci run -s SES_4XJ3K "..."                              # a specific session
synsci session show SES_4XJ3K
```

| Command | Use |
| --- | --- |
| `synsci run -c` | Continue the most recent session in the current directory. |
| `synsci run -s <id>` | Resume a specific session by id. |
| `synsci session list` | List recent sessions for the account. |
| `synsci session show <id>` | Print prompts, replies, and tool calls. |

## Attach to a running agent

Leave `synsci web` running and attach a TUI from another terminal instead of starting fresh:

```bash
synsci tui attach           # attach to the running local agent
synsci tui thread <id>      # attach and open a specific thread
```

This drives a long-running local agent from a different terminal.

## Output

```bash
synsci run --format json "Summarize the architecture of src/" > arch.jsonl
synsci run --title "src arch" "Summarize..."
```

JSON output emits one envelope per event (prompt, reply, tool call, tool result, completion). Parse it line-by-line or capture the final envelope. `--title` sets the title in `synsci session list`.

## Export and import

```bash
synsci export SES_4XJ3K --out session.json
synsci import session.json
```

`export` writes a portable JSON document with the full session; `import` accepts it on another machine. Useful for sharing reproductions or attaching a session to an issue.

## Session vs profile vs skill

- A **session** is one conversation, with an id, transcript, and working directory.
- An **agent profile** is the system prompt and tool set the agent uses. See [Sub-agents](/agent-cli/sub-agents).
- A **skill** is a portable instruction bundle layered on top. See [Skills](/agent-cli/skills).

The same profile and skills drive thousands of sessions; sessions are the cheap, disposable unit.
