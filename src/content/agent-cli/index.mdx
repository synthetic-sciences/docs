**Atlas Agent CLI** is the SynSci agent CLI — the same product, now part of the Atlas suite. It is a terminal-first coding agent built for ML engineering and scientific research that records its work straight into [Atlas Graphs](/graphs/index) and grounds itself in [Atlas Library](/library/index).

<Warning>
"Atlas Agent CLI" is the new name for the SynSci agent CLI — it is one product, not two. It ships today on npm as `synsci`; that package name predates the Atlas branding and is not changing yet, so you still install it with `npm i -g synsci`. The in-app Agent surface is rolling out and currently shows "coming soon".
</Warning>

## How it fits the suite

The agent authenticates with your Atlas account and uses the same `thk_*` credentials as the rest of the suite — there is no separate pairing step. As it works, it records runs, evidence, and decisions into the graph through Atlas, so an agent session leaves the same durable trail a human would.

```text
you / a schedule
  -> synsci  (the SynSci agent CLI)
  -> model provider (BYOK or managed route)
  -> tool calls (files, shell, skills) on your machine
  -> runs + decisions recorded into Atlas Graphs
```

## Run the agent

```bash
npm i -g synsci
synsci                      # open the TUI agent in the current repo
synsci run "Refactor utils/log.ts to use structured logging"
synsci web                  # local browser UI on 127.0.0.1:4444
```

It is not a sandbox — it has the same filesystem and shell access you do — so run it in a project you trust or inside a container. See [Security](/agent-cli/security).

## Vocabulary

- A **session** is one agent conversation with its own history, tool calls, and working directory.
- A **provider** is a model backend the agent routes through (BYOK or the managed route).
- An **agent profile** is a system prompt plus a tool set plus a routing policy.
- A **skill** is a portable instruction bundle that primes the agent for a domain.
