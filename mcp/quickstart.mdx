The [`@synsci/thesis`](https://www.npmjs.com/package/@synsci/thesis) setup helper handles the entire connection process for you. It scans your machine for supported agent hosts, opens a browser to authenticate your Thesis account, creates a scoped API key, and writes the MCP server entry into each detected host's config file, all in a single run.

<Note>
  `@synsci/thesis` is a **setup helper** for the Thesis MCP server. It is not the same npm package as [`@synsci/cli`](/cli/introduction), which is the standalone Synsci CLI terminal coding agent. If you're using Synsci CLI, see [Connect Synsci CLI to Thesis](/cli/mcp) instead, the registration path is different.
</Note>

## Run the setup command

<Steps>
  <Step title="Run setup">
    Open a terminal and run the setup command with your preferred package manager. No global install required.

    <CodeGroup>

    ```bash npx
    npx --yes @synsci/thesis setup
    ```

    ```bash bunx
    bunx @synsci/thesis setup
    ```

    ```bash pnpx
    pnpx @synsci/thesis setup
    ```

    ```bash yarn dlx
    yarn dlx @synsci/thesis setup
    ```

    </CodeGroup>
  </Step>

  <Step title="Host detection">
    The CLI scans for installed agent hosts by checking known config paths on disk. It automatically detects Claude Code, OpenCode, Codex, Cursor, Pi, Hermes Agent, and OpenClaw.

    If you want to target a specific host instead of all detected ones, pass the `--hosts` flag:

    ```bash
    npx --yes @synsci/thesis setup --hosts cursor
    ```
  </Step>

  <Step title="Authenticate">
    Your browser opens to the Thesis OAuth login page. Sign in with your Thesis account to continue.

    If you're on a remote or headless machine where a browser cannot open, use device-code authentication instead:

    ```bash
    npx --yes @synsci/thesis setup \
      --auth-mode device \
      --base-url https://thesis.syntheticsciences.ai
    ```

    The CLI prints a short code and URL. Open the URL on any device, enter the code, and the CLI continues automatically.
  </Step>

  <Step title="API key provisioned">
    After authentication, the CLI creates a Thesis API key scoped to your account. The key uses the `thk_*` prefix and is stored automatically, you do not need to copy or save it manually.
  </Step>

  <Step title="MCP config written">
    The CLI injects the Thesis MCP server entry into the config file for each detected host. No manual JSON or YAML editing required.
  </Step>
</Steps>

## Web-based hosts

Claude.ai and ChatGPT.com do not read local config files. To connect Thesis to these hosts, use the remote MCP server URL when the host prompts for an MCP server address:

```
https://thesis.syntheticsciences.ai/mcp-server
```

You will also need a Thesis API key to authenticate. See [Authentication](/mcp/authentication) for how to retrieve your key.

## Uninstall

To remove the Thesis MCP server entry from all detected host configs, run:

```bash
npx --yes @synsci/thesis uninstall
```

<Note>
  The uninstall command only removes the MCP config entries. It does not delete your Thesis account, API keys, or research data.
</Note>
