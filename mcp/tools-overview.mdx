Thesis exposes 86 tools through a single JSON-RPC 2.0 endpoint at `/mcp-server`. Every tool is available to any MCP-compatible agent host. Claude Code, Cursor, Codex, OpenCode, or your own client, using a Bearer token for authentication. Tools are grouped into 16 functional categories covering the full research lifecycle, from creating and branching nodes to leasing GPU compute, exporting subgraphs, and auditing activity.

## Calling convention

All requests are HTTP POST to `/mcp-server` with `Content-Type: application/json` and an `Authorization: Bearer <token>` header. The body is a standard JSON-RPC 2.0 envelope with `method` set to `tools/call`.

<ParamField body="jsonrpc" type="string" required default='"2.0"'>
  Must be `"2.0"`.
</ParamField>

<ParamField body="id" type="number | string" required>
  Client-generated request identifier. Echoed back in the response.
</ParamField>

<ParamField body="method" type="string" required default='"tools/call"'>
  Always `"tools/call"` for tool invocations.
</ParamField>

<ParamField body="params.name" type="string" required>
  The tool name, e.g. `thesis_stage_node_create`.
</ParamField>

<ParamField body="params.arguments" type="object" required>
  Tool-specific arguments. Schema varies per tool; see the individual tool pages.
</ParamField>

### Example: create a node

<CodeGroup>

```json Request
POST /mcp-server
Content-Type: application/json
Authorization: Bearer thk_your_api_key

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "thesis_stage_node_create",
    "arguments": {
      "title": "Attention vs. convolution on long sequences",
      "summary": "Test whether attention outperforms convolution past 8k tokens",
      "content": "Run both architectures on synthetic long-range dependency tasks. Vary sequence length from 1k to 32k. Measure accuracy and wall-clock time.",
      "kind": "empirical",
      "parent_ids": ["a1b2c3d4-0000-0000-0000-000000000001"]
    }
  }
}
```

```json Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"node\": {\"node_id\": \"e5f6a7b8-0000-0000-0000-000000000042\", \"revision\": 0, \"kind\": \"empirical\", \"title\": \"Attention vs. convolution on long sequences\", \"lifecycle\": \"staged\", \"parent_ids\": [\"a1b2c3d4-0000-0000-0000-000000000001\"]}}"
      }
    ]
  }
}
```

</CodeGroup>

The `result.content` array always contains a single `text` element whose `text` field is a JSON-encoded object. Parse that string to access the returned data.

## Idempotency

Mutation tools in Thesis are automatically idempotent. When you call a mutation tool, the MCP server computes a SHA-256 key from the canonical JSON serialization of your parameters. If you send the exact same parameters again within 7 days, the server replays the cached response without re-executing the operation. This protects against duplicate writes caused by network retries or agent restarts.

<Note>
  Idempotency is keyed on the full parameter set. Changing any argument, even whitespace in a string field, produces a new key and triggers a fresh execution.
</Note>

## Tool categories

The 86 tools are grouped into 16 categories. Mutation tools (those that write or modify state) are marked in each category's reference page.

| Category | Count | What it covers |
|---|---|---|
| [Nodes](/mcp/tools-nodes) | 15 | Create, update, commit, branch, merge, and query research nodes |
| [Sharing](/mcp/tools-nodes) | 3 | Set per-node or batch sharing modes and collaborators |
| [Tags](/mcp/tools-nodes) | 4 | Create, rename, and assign tags to nodes |
| [Artifacts](/mcp/tools-nodes) | 7 | Upload, retrieve, preview, and delete node file attachments |
| [Knowledge](/mcp/tools-knowledge) | 9 | Index papers, repos, docs, and datasets; search and read sources |
| [Oracle](/mcp/tools-knowledge) | 4 | Queue and stream long-running research synthesis jobs |
| [Tracer](/mcp/tools-knowledge) | 2 | GitHub code search across repositories |
| [Document Agent](/mcp/tools-knowledge) | 1 | Q&A against a single indexed source |
| [Compute](/mcp/tools-compute) | 11 | Lease GPU instances and manage approval sessions |
| [Workflow](/mcp/tools-compute) | 7 | Start, progress, and complete multi-step research workflows |
| [Campaigns](/mcp/tools-compute) | 5 | Manage campaign budgets and snapshots |
| [Execution](/mcp/tools-compute) | 3 | Launch, list, and terminate code executions |
| [Auth & Meta](/mcp/tools-compute) | 4 | Check auth status, credits, and system contract |
| [Export/Import](/mcp/tools-export) | 6 | Export subgraphs as JSON, markdown, or PDF; import subgraphs |
| [Updates](/mcp/tools-export) | 4 | List and manage in-app notifications |
| [Audit](/mcp/tools-export) | 1 | Retrieve audit events for a node |

- **[Node and graph tools](/mcp/tools-nodes).** Create, branch, commit, merge, tag, share, and upload artifacts to nodes.
- **[Knowledge and search tools](/mcp/tools-knowledge).** Index sources, run unified search, launch Oracle jobs, and query documents.
- **[Compute and workflow tools](/mcp/tools-compute).** Acquire GPU leases, run workflows, manage budgets, and launch executions.
- **[Export, import, and audit tools](/mcp/tools-export).** Export subgraphs, generate PDF/markdown summaries, and review audit trails.
