---
title: "Thesis skills: reusable workflows for your agent hosts"
description: "Thesis skills guide your agent host through Thesis workflows, survey knowledge, graphify sources, validate claims, or run autonomous frontier advancement."
---

Thesis skills are prebuilt instruction sets you install alongside the Thesis MCP server. Each skill tells your agent host exactly how to drive Thesis for a specific research task, when to create nodes, how to persist a run contract, which tools to call, and when to stop. Rather than writing these workflows yourself, you install a skill and invoke it by name inside your agent host.

## Two variants per skill

Every skill ships in two flavors. Choose the one that fits how your agent host works.

<Tabs>
  <Tab title="MCP variant">
    The MCP variant exposes Thesis through the 86 structured tool schemas registered at `/mcp-server`. Your agent host calls `thesis_*` tools directly, node creation, artifact uploads, compute approval, search, and more.

    **Best for:** hosts that exploit structured tool calls natively (Cursor, OpenCode, Pi, Hermes Agent, OpenClaw).

    Install with:

    ```bash
    npx --yes @synsci/thesis setup --install-skill
    ```
  </Tab>
  <Tab title="CLI variant (−cli)">
    The CLI variant (`-cli` suffix) drives Thesis through bundled bash subcommands instead of MCP tool schemas. Your agent invokes shell scripts; no MCP schema overhead lands in its context window.

    **Best for:** hosts with strong shell affordances where you want to keep context focused on research rather than tool definitions (Claude Code and similar).

    Install with:

    ```bash
    npx --yes @synsci/thesis setup --install-skill
    ```

    Then provision credentials for the bash scripts:

    ```bash
    npx --yes @synsci/thesis setup --cli --api-key <KEY>
    ```

    This writes `~/.config/thesis/env` once. Every `-cli` skill sources it on each invocation.
  </Tab>
</Tabs>

<Note>
  Both variants talk to the same Thesis backend and produce the same durable graph, log, and artifact state. The choice is about how your agent host prefers to receive tool definitions, not about capability.
</Note>

## When to use MCP vs CLI

| Situation | Recommended variant |
|-----------|---------------------|
| Host natively handles structured JSON tool schemas | MCP |
| Host has strong shell affordances (e.g. Claude Code) | CLI (`-cli`) |
| You want to minimize schema tokens in the context window | CLI (`-cli`) |
| You need to customize bash scripts after install | CLI (`-cli`) |
| Host is a web connector (Claude.ai, ChatGPT.com) | MCP |

## The six skill families

- **thesis / thesis-cli.** General Thesis usage, experiment design, setup support, and troubleshooting across supported hosts. The entry-point skill for platform questions and cross-cutting guidance.
- **thesis-auto / thesis-auto-cli.** Advance a research frontier autonomously under an explicit budget. Persists a run contract in the graph, keeps the plan `n` hops ahead, and replans after each resolution.
- **thesis-lookahead / thesis-lookahead-cli.** Stage the next-step frontier plan from existing graph state without executing it. Pure planning: no compute approval, no execution, no spend.
- **thesis-reproduce / thesis-reproduce-cli.** Graphify claim-bearing material and validate it empirically within a hard maximum budget. Combines source-to-graph authoring with budgeted validation branches.
- **thesis-survey / thesis-survey-cli.** Discover, search, and synthesize relevant knowledge from indexed sources and the Thesis graph into committed insight nodes with citations.
- **thesis-to-graph / thesis-to-graph-cli.** Port source material, papers, READMEs, wikis, research notes, into Thesis nodes and artifacts without execution or compute spend.


## Installing skills

Run the standard setup command with `--install-skill` to install both MCP and CLI variants for all six families:

```bash
npx --yes @synsci/thesis setup --install-skill
```

The installer detects your agent hosts, injects the Thesis MCP config, copies the bundled bash scripts into each `-cli` skill directory, and leaves each skill self-contained.

<Tip>
  If you only want skills for a specific host, pass `--hosts <name>` to target it. For example, `--hosts claude` installs only into the Claude Code config.
</Tip>

## How skills are loaded

Skills are installed into your agent host's skill or slash-command directory. When you invoke a skill by name inside your host, the host loads the corresponding `SKILL.md` and reference files into context. The skill then guides your agent through the Thesis workflow for that task.

The `-cli` variant skills include bundled bash scripts (copied at install time into the skill directory) so the skill is fully self-contained, no separate PATH entries required.
