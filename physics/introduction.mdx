Synsci handles computational physics workflows from quantum simulation through classical fluid dynamics. Activate the physics mode and the agent treats your prompts the way a computational physicist would: it picks the right solver, the right discretization, and the right hardware.

## Activate the physics mode

```bash
synsc web --mode physics
```

For a one-shot run:

```bash
synsc run --mode physics "Set up an incompressible Navier-Stokes solve on this mesh and measure drag"
```

To set physics as the default for your current working directory:

```bash
synsc mode set physics
```

## Who this is for

- Computational physicists working in CFD, lattice methods, or N-body
- Quantum information researchers running circuit simulations and noise studies
- Plasma physicists doing PIC, MHD, or kinetic simulations
- Astrophysicists running galaxy and stellar dynamics
- Engineers doing structural FEA and acoustics

## What you can do

- Quantum circuit simulation and noise modeling
- Classical and incompressible fluid dynamics
- Particle-in-cell plasma simulations
- N-body astrophysics and orbital integration
- Lattice QCD and lattice gauge theory
- Finite element analysis for structures and electromagnetics

## Frameworks the agent uses

When you describe a physics task, the agent reaches for these natively. You don't need to invoke them by name.

| Category | Stack |
|---|---|
| Quantum simulation | Qiskit, Cirq, PennyLane, TensorNetwork, Stim |
| Computational fluid dynamics | OpenFOAM, FEniCS, Nek5000, Basilisk |
| N-body and astrophysics | REBOUND, AMUSE, GADGET-4 |
| Lattice methods | Chroma, Grid, MILC for lattice QCD |
| Plasma physics | WarpX, Smilei, PIConGPU |
| Finite element | FEniCS, Firedrake, deal.II |
| Visualization | ParaView, VisIt, yt |
