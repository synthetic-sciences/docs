Each recipe shows what to type into the agent and what comes out the other side. Use them as starting points and adapt to your system.

## DFT geometry optimization

> Optimize this MOF structure with PBE-D3 in VASP. Use a 4x4x4 k-point grid and converge forces below 0.01 eV/A. Report the band gap when done.

What happens:

1. The agent reads your structure (CIF, POSCAR, or XYZ).
2. Generates `INCAR` with PBE-D3 settings and the requested convergence tolerances.
3. Sets up `KPOINTS` with the 4x4x4 grid.
4. Submits the job locally, or on Modal/Lambda for larger systems.
5. Streams VASP output and reports convergence and the band gap when finished.

## Molecular dynamics simulation

> Set up a GROMACS simulation of this protein-ligand complex. Use AMBER ff14SB plus GAFF2 for the ligand. Equilibrate 5 ns NVT then 5 ns NPT, then 100 ns production at 300 K.

What happens:

1. Generates topology files for both protein and ligand.
2. Sets up the equilibration phases with appropriate restraints.
3. Configures the production run with output frequency tuned for analysis.
4. Submits the simulation to compute (Modal or Lambda based on system size).
5. Pulls back trajectory data and runs basic analysis (RMSD, RMSF, hydrogen bonds).

## Drug docking and virtual screening

> Dock this 50,000-compound library against the active site of 1abc.pdb using GNINA. Filter for predicted affinity better than -8 kcal/mol and write the top 100 to results/.

What happens:

1. Retrieves the protein structure and identifies the binding site.
2. Prepares the receptor (adds hydrogens, assigns charges).
3. Runs GNINA across the library, parallelizing across GPU nodes on Modal.
4. Filters by predicted affinity, ranks the survivors, and writes the top 100 with their poses.

## Retrosynthesis planning

> Plan retrosynthetic routes for ibuprofen using a transformer-based model. Rank by predicted yield and feasibility, and surface the top three routes with reaction conditions.

What happens:

1. The agent uses a transformer retrosynthesis model.
2. Generates candidate retrosynthetic trees with multiple disconnection options.
3. Ranks routes by predicted yield, step count, and reagent availability.
4. Returns the top routes with conditions, reagents, and predicted yields per step.

## Excited state calculation

> Compute the first 10 singlet and triplet excited states of this molecule with TDDFT (CAM-B3LYP / def2-TZVP) in PySCF. Report oscillator strengths and the dominant orbital transitions.

What happens:

1. The agent loads the molecule into PySCF.
2. Runs the ground state SCF.
3. Runs TDDFT with the requested functional and basis.
4. Reports excitation energies, oscillator strengths, and dominant transition character for each root.

## Cheminformatics analysis

> Analyze this dataset of 50,000 ChEMBL compounds. Filter for Lipinski rule-of-five compliance, compute Murcko scaffolds, and cluster by Tanimoto similarity. Return cluster summary statistics and the 10 most common scaffolds.

What happens:

1. Loads the dataset with RDKit.
2. Applies Lipinski filters and reports how many compounds pass.
3. Extracts Murcko scaffolds.
4. Computes Morgan fingerprints and Tanimoto similarity matrix.
5. Clusters with Butina or hierarchical clustering and returns cluster sizes and the most common scaffolds.

## Free energy perturbation

> Run a relative binding free energy calculation between these two ligands against PDB 5abc using OpenFE. Use 12 lambda windows per leg and 5 ns per window.

What happens:

1. Sets up the alchemical transformation between the two ligands.
2. Generates the network of perturbations.
3. Runs the windows in parallel on Modal GPUs.
4. Analyzes the simulations with MBAR.
5. Reports the relative ΔΔG with statistical uncertainty.
