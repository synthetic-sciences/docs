Synsci CLI is built around research modes. Each mode is a self-contained working environment for a scientific domain. Setting a mode primes the agent with the right context, activates the bundled skills for that domain, and routes the work to the backend best suited for it. You stay focused on the research; the CLI handles the rest.

## Why modes

A computational chemist asking about DFT and a machine learning engineer asking about GRPO want very different things from the same prompt. Modes solve this by giving the agent enough scaffolding upfront that it doesn't have to guess. When you tell `synsc` you're working in chemistry, the agent assumes chemistry conventions, uses chemistry tooling, and reads files through chemistry skills.

## Available modes

### Chemistry

Computational chemistry, drug discovery, reaction modeling, and materials chemistry.

Bundled skills cover:

- DFT and ab initio calculations (PySCF, ORCA, NWChem, Psi4)
- Molecular dynamics (GROMACS, OpenMM, LAMMPS, AMBER)
- Cheminformatics and reaction prediction (RDKit, Open Babel, retrosynthesis transformers)
- Drug discovery (AutoDock Vina, GNINA, DiffDock, equibind)
- Quantum chemistry workflows (CASSCF, CCSD(T), TDDFT)

Activate the mode:

```bash
synsc web --mode chemistry
synsc --mode chemistry
```

Try a prompt like:

> Optimize this MOF structure with PBE-D3, then compute the band gap.

### Biology

Genomics, structural biology, single-cell analysis, and computational biology more broadly.

Bundled skills cover:

- Protein structure prediction (AlphaFold 3, ESMFold, RoseTTAFold, OpenFold)
- Sequencing pipelines (samtools, BWA, GATK, DeepVariant)
- Single-cell and spatial transcriptomics (Scanpy, Seurat, Squidpy)
- Multi-omics integration (MOFA, Harmonypy, scVI)
- Phylogenetics and evolution (IQ-TREE, RAxML, BEAST)

Activate the mode:

```bash
synsc web --mode biology
```

Try a prompt like:

> Predict structures for these 12 protein sequences with ESMFold and rank by pLDDT.

### Physics

Computational physics, simulations, quantum systems, and high-performance numerics.

Bundled skills cover:

- Quantum simulation (Qiskit, Cirq, PennyLane, TensorNetwork)
- Computational fluid dynamics (OpenFOAM, FEniCS, Nek5000)
- N-body and astrophysics (REBOUND, AMUSE, GADGET)
- Lattice QCD and lattice gauge theory
- Plasma physics and PIC codes

Activate the mode:

```bash
synsc web --mode physics
```

Try a prompt like:

> Set up a 2D incompressible Navier-Stokes solve on this mesh and measure drag.

### Materials science

Solid-state materials, crystal structure, and materials engineering.

Bundled skills cover:

- DFT for materials (VASP, Quantum ESPRESSO, GPAW, ABINIT)
- Crystal structure prediction (CALYPSO, USPEX, AIRSS)
- High-throughput materials discovery (pymatgen, atomate, jarvis-tools)
- Phase diagrams and CALPHAD
- Machine-learning interatomic potentials (NequIP, MACE, Allegro)

Activate the mode:

```bash
synsc web --mode materials
```

Try a prompt like:

> Use MACE to relax this perovskite supercell, then compute phonons with phonopy.

### Machine learning

Training, fine-tuning, evaluation, inference, and deployment.

Bundled skills cover:

- Training frameworks (PyTorch DDP, FSDP, DeepSpeed, Megatron, Axolotl)
- RLHF and post-training (GRPO, PPO, DPO, RLOO, KTO)
- Inference and serving (vLLM, TensorRT-LLM, SGLang, llama.cpp)
- Evaluation harnesses (lm-eval-harness, BIG-bench, HELM)
- Fine-tuning (LoRA, QLoRA, ReFT, full-parameter SFT)

Activate the mode:

```bash
synsc web --mode ml
```

Try a prompt like:

> Fine-tune llama-3-8b with QLoRA on this dataset, log to wandb, and serve with vLLM.

### Mathematics

Symbolic math, theorem proving, optimization, and computational mathematics.

Bundled skills cover:

- Symbolic computation (SymPy, SageMath, Mathematica integration)
- Theorem proving (Lean 4, Coq, Isabelle/HOL)
- Numerical analysis (NumPy, SciPy, JAX patterns)
- Convex and integer optimization (CVXPY, JuMP, Gurobi)
- Computer algebra systems

Activate the mode:

```bash
synsc web --mode math
```

Try a prompt like:

> Formalize this proof in Lean 4 and check it compiles.

### General

The default mode when no domain is specified. Standard coding agent capabilities without research-specific priming. Use it for general engineering work, infrastructure, or anything outside the bundled domains.

## Switching modes

Mid-session, use the slash command:

```
/mode chemistry
```

Per-launch, pass the flag:

```bash
synsc web --mode biology
```

Set a default for a working directory so subsequent sessions start in the right mode:

```bash
synsc mode set chemistry
```

The active mode persists across sessions through the dashboard, so resuming a session in a fresh shell picks up where you left off.

## What setting a mode actually does

Three things happen when a mode becomes active.

1. The agent's system context gets a domain primer. It now knows the conventions, tooling, and norms of that field.
2. The bundled skills for that domain become trigger-active. Other skills stay loadable on demand but aren't surfaced first.
3. The CLI routes the work to the backend best suited for the domain and task. You don't pick a provider or a model. The system handles it inside the credit envelope you're already on.

## Inspecting modes

List what's available on your install:

```bash
synsc mode list
```

Show what's active in the current session:

```bash
synsc mode show
```

Switch to general mode (the default, no priming):

```bash
synsc mode set general
```
