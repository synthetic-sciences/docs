Skills are the CLI's domain knowledge. Each skill is a self-contained playbook plus optional helper scripts that teach the agent how to do one thing well. The active [research mode](/cli/modes) determines which skills surface first. The rest stay loadable on demand if your prompt drifts into them.

## What a skill covers

A skill is short and focused: how to start a job, what flags to pass, common pitfalls, where outputs land, and how to read failure modes. Skills span every supported research mode.

### Chemistry skills

DFT and ab initio (PySCF, ORCA, NWChem, Psi4), molecular dynamics (GROMACS, OpenMM, LAMMPS, AMBER), cheminformatics (RDKit, Open Babel), drug discovery (AutoDock Vina, GNINA, DiffDock), and quantum chemistry workflows.

### Biology skills

Protein structure prediction (AlphaFold 3, ESMFold, RoseTTAFold), sequencing pipelines (samtools, BWA, GATK, DeepVariant), single-cell and spatial transcriptomics (Scanpy, Seurat), multi-omics integration, and phylogenetics.

### Physics skills

Quantum simulation (Qiskit, Cirq, PennyLane), CFD (OpenFOAM, FEniCS, Nek5000), N-body and astrophysics (REBOUND, AMUSE), lattice QCD, and plasma physics.

### Materials skills

DFT for materials (VASP, Quantum ESPRESSO, GPAW, ABINIT), crystal structure prediction, high-throughput discovery (pymatgen, atomate, jarvis-tools), CALPHAD, and ML interatomic potentials (NequIP, MACE, Allegro).

### Machine learning skills

Training (PyTorch DDP, FSDP, DeepSpeed, Megatron, Axolotl), RLHF (GRPO, PPO, DPO, RLOO, KTO), inference (vLLM, TensorRT-LLM, SGLang, llama.cpp), evaluation (lm-eval-harness, BIG-bench, HELM), and fine-tuning (LoRA, QLoRA, ReFT, full SFT).

### Mathematics skills

Symbolic computation (SymPy, SageMath), theorem proving (Lean 4, Coq, Isabelle), numerical analysis, and convex/integer optimization (CVXPY, JuMP).

<Note>
  The full list is exposed in the CLI itself. Run `synsc` to start a session, then ask: "List the bundled skills you have access to in this mode." The agent enumerates what's currently active.
</Note>

## How skills load

Skills are not always-on. The agent loads a skill into context only when your prompt is on-topic. This keeps the model context lean even though the binary ships hundreds of skills across all the modes.

A typical flow:

1. You're in `chemistry` mode and ask: "Optimize this MOF structure with PBE-D3, then compute the band gap."
2. The agent matches the prompt against the chemistry skills, finds the DFT playbook for VASP/Quantum ESPRESSO, and loads it.
3. The skill tells the agent how to scaffold the input file, what convergence criteria matter, where to point at the structure, and what to log.
4. The agent executes. Writing files, kicking off the run, streaming logs.
5. Once the task is done or you change topic, the skill unloads.

You don't manage the lifecycle. The agent handles it.

## Trigger phrases

You don't need to invoke skills by name. Just describe the task in normal language and the agent picks the right skill from the active mode.

| Mode | If you ask, | The agent loads, |
|---|---|---|
| chemistry | "Run DFT optimization on this MOF with PBE-D3" | VASP / Quantum ESPRESSO DFT skill |
| biology | "Predict structures for these protein sequences" | AlphaFold / ESMFold skill |
| physics | "Set up an incompressible Navier-Stokes solve on this mesh" | OpenFOAM / FEniCS skill |
| materials | "Relax this perovskite supercell with MACE" | MACE + phonopy skill |
| ml | "Fine-tune llama-3-8b with QLoRA on this dataset" | LoRA / QLoRA / transformers skill |
| math | "Formalize this proof in Lean 4 and check it compiles" | Lean 4 skill |

## Skills and credentials together

Skills know about the credentials [synced from your dashboard](/cli/credentials). When the GRPO skill needs to log to Weights & Biases, it uses the W&B token already in your environment. When the Modal skill spins up a sandbox, it uses your Modal token. When `from datasets import load_dataset` needs Hugging Face auth, your HF token is there.

## When a bundled skill isn't enough

If you need a workflow that isn't covered by a bundled skill:

- Describe it in detail in your prompt. The agent is capable enough to scaffold most jobs from a careful description.
- Add it to the skills repo. The Synsci skills library is open and accepts contributions. The maintained library lives at [github.com/SyntheticSciences/AI-research-SKILLs](https://github.com/SyntheticSciences/AI-research-SKILLs).
- Use a custom skill in your project. Drop a `.synsc/skills/<name>.md` file in your repo and the agent will load it alongside the bundled skills for sessions started in that directory.
