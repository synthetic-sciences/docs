`synsc web` is the recommended way to use Synsci CLI day-to-day. It boots a local server on your machine and opens a browser tab with a full chat interface — multi-tab sessions, side-by-side file diffs, command palette, and the same agent surface as the TUI. No separate desktop install, no leaving your browser, and no eating a terminal split.

## Start the web UI

```bash
synsc web
```

The CLI prints the local URL and opens your default browser to it. The default port is `4444`; pass `--port` to override:

```bash
synsc web --port 5555
```

The server runs as long as the `synsc web` process is alive. Stop it with <kbd>Ctrl+C</kbd> in the terminal where you started it, or close the browser tab and let the idle timeout shut it down.

## Why pick web over TUI or desktop

| Situation | Pick |
|---|---|
| Daily development, multi-hour sessions | **Web** |
| Quick edits, scripts, CI hooks | TUI (`synsc` or `synsc run`) |
| You prefer a native app over a browser tab | [Desktop](/cli/desktop) |
| Headless or remote-only machine | TUI (no browser available) |
| Pair programming, screen-sharing demos | **Web** (cleaner UI than TUI) |

The web UI shares the same backend as the TUI and the desktop app, so model usage, credentials, and credits all flow through the same dashboard account. Switching between modes mid-day is fine — sessions started in one mode show up in the dashboard alongside sessions from the others.

## Switching agents and models

In the web UI, agent and model controls live in the header:

- **Agent toggle** — flip between **build** and **plan** with one click.
- **Model picker** — pick Claude Opus 4, Sonnet 4.5, GPT Codex Mini, or Gemini 2.5 Pro for the current session.

Mid-session changes apply to the next message. Earlier messages stay in context with whatever model produced them.

You can also use slash commands inside the chat:

```
/agent plan
/model claude-opus-4
/clear
/save
/help
```

## Keyboard shortcuts

| Shortcut | Action |
|---|---|
| <kbd>Cmd+K</kbd> | Open command palette |
| <kbd>Cmd+T</kbd> | New session tab |
| <kbd>Cmd+W</kbd> | Close current tab |
| <kbd>Cmd+L</kbd> | Focus the chat input |
| <kbd>Cmd+,</kbd> | Open preferences |

On Windows and Linux, substitute <kbd>Ctrl</kbd> for <kbd>Cmd</kbd>.

## Remote access

`synsc web` binds to `127.0.0.1` by default — only your local machine can reach it. To expose the UI on your LAN (for example, to use it from a tablet or another machine), pass `--host`:

```bash
synsc web --host 0.0.0.0 --port 4444
```

<Warning>
  Binding to `0.0.0.0` makes the agent reachable from anyone on your network. Don't do this on shared or untrusted networks. The web UI requires a Bearer token (issued at session start), but exposing it broadens the attack surface — use SSH port-forwarding instead if you need remote access:

  ```bash
  ssh -L 4444:127.0.0.1:4444 your-remote-host
  ```
</Warning>

## When `synsc web` isn't right

A few cases where you'll reach for something else:

- **`synsc run "<prompt>"`** — one-shot scripted runs without any UI.
- **`synsc` (bare TUI)** — when you want to stay in the terminal entirely.
- **[Desktop app](/cli/desktop)** — native windowed app with the same agent surface.

All three share the same dashboard account, credentials, and credits.
