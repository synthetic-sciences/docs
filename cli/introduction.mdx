Synsci CLI is a terminal coding agent built for scientific work. The binary is `synsc`. It runs alongside your shell and gives you a single agent that already knows how to do science. Set the research mode (chemistry, biology, physics, materials, machine learning, or mathematics) and the agent loads domain-aware tools, cloud credentials, and the right backend automatically.

To install, run `npx synsci`. The npm package is [`@synsci/cli`](https://www.npmjs.com/package/@synsci/cli). The dashboard lives at [cli.syntheticsciences.ai](https://cli.syntheticsciences.ai).

<Note>
  Synsci CLI is a separate product from [Thesis](/introduction). Thesis is the research workspace (graphs, experiments, MCP server). Synsci CLI is the agent you run in your shell. They pair via MCP if you use both.
</Note>

## What you get

- **Research modes** for [chemistry](/chemistry/introduction), [biology](/biology/introduction), [physics](/physics/introduction), [materials](/materials/introduction), [machine learning](/ml/introduction), and [mathematics](/math/introduction). Each mode primes the agent with domain conventions and routes calls to the right backend.
- **Bundled domain tools** for each mode. From DFT and molecular dynamics in chemistry to GRPO and vLLM in ML. The agent loads them automatically when your prompt calls for them.
- **[Credential sync](/cli/credentials)** for Hugging Face, Weights & Biases, Modal, Lambda, AWS, and GCP. Tokens come from your dashboard and reach subprocesses without ever touching disk.
- **[Included credits](/cli/credits)** that cover the agent's work. Pre-call balance checks block runaway loops before they bill.

## Two agents, one toggle

Two top-level agents are available in every UI:

- `build` is the default. Full filesystem and shell access for the actual research work.
- `plan` is read-only. Use it for analysis, code exploration, and design where you don't want anything written.

Switch with the agent toggle in the [web UI](/cli/web) header, or press <kbd>Tab</kbd> in the TUI. Mention `@general` in any message to delegate a search or multi-step task to a sub-agent that runs in its own context.

## Where it fits

| You want to | Reach for |
|---|---|
| Run DFT, MD, or other compute work in a research domain | Synsci CLI with the right research mode |
| Track hypotheses, experiments, and decisions over time | [Thesis](/introduction) |
| Use both, with the agent writing code and Thesis storing what was tried | Run both, paired through [MCP](/cli/mcp) |
