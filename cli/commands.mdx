This page lists every subcommand that ships with `synci`, what it does, and the most useful flags. The bare `synci` (no arguments) drops you into the TUI in the build agent and the `general` research mode.

## Connection

### `synci connect`

Manage the Bearer token connection between your CLI and the dashboard at [cli.syntheticsciences.ai](https://cli.syntheticsciences.ai).

| Subcommand | Purpose |
|---|---|
| `synci connect login` | Run the device-flow OAuth login. Opens your browser and stores the resulting Bearer token in OS keychain. |
| `synci connect status` | Show your account email, synced credentials, active mode, and credit balance. |
| `synci connect logout` | Invalidate the local Bearer token. |

### `synci codex`

Sign in to OpenAI Codex from the CLI. After the browser flow completes, the resulting Codex token is held in the OS keychain alongside your Synci Bearer token, and is injected into Codex-aware subprocesses (`codex`, `codex exec`, etc.) on demand.

| Subcommand | Purpose |
|---|---|
| `synci codex login` | Open the Codex sign-in flow in your browser and store the returned token. |
| `synci codex status` | Show the active Codex account, token expiry, and which workspaces are reachable. |
| `synci codex logout` | Revoke and remove the local Codex token. |

```bash
synci codex login
synci codex login --workspace my-org
synci codex status
```

Flags:

- `--workspace <slug>` to bind the token to a specific Codex workspace (default: your personal workspace).
- `--no-browser` to print a device-code URL instead of opening a browser; useful on headless boxes.

## Sessions

### `synci web` (recommended)

Boot a local server and open the browser UI. The recommended day-to-day interface. See [Web UI](/cli/web) for the full feature set.

```bash
synci web
synci web --mode chemistry
synci web --port 5555
synci web --agent plan
synci web --host 0.0.0.0    # expose to LAN, use carefully
```

Flags:

- `--mode <name>` for the research mode (research, physics, chemistry, biology, write, flywheel).
- `--agent <build|plan>` for the agent (default: `build`).
- `--port <n>` to bind on `127.0.0.1` (default: `4444`).
- `--host <addr>` to override the bind interface (default: `127.0.0.1`).
- `--no-open` to skip auto-opening the browser.

### `synci` (no args, TUI mode)

Start a TUI session. Useful when you want everything in the terminal.

```bash
synci
synci --mode biology
synci --agent plan
synci --project research-fork
```

Flags:

- `--mode <name>` for the research mode.
- `--agent <build|plan>` for the agent.
- `--project <name>` to tag the session for dashboard rollups.
- `--cwd <path>` to start with a specific working directory.

### `synci run`

One-shot run. Sends the prompt, lets the agent work to completion, and exits.

```bash
synci run --mode ml "Refactor utils/log.ts to use structured logging"
synci run --mode chemistry "Set up a DFT optimization for this CIF file"
synci run --agent plan "Summarize the architecture of src/"
```

Useful for git hooks, CI, and shell pipelines.

### `synci session`

Inspect and manage past sessions stored on the dashboard.

| Subcommand | Purpose |
|---|---|
| `synci session list` | List recent sessions. |
| `synci session show <id>` | Print the prompts and assistant messages for a session. |
| `synci session resume <id>` | Continue a prior session in the TUI. |

## Research modes

### `synci mode`

Manage research modes. See [research](/cli/research/research), [physics](/cli/research/physics), [chemistry](/cli/research/chemistry), [biology](/cli/research/biology), and [flywheel](/cli/research/flywheel) for what each mode covers.

| Subcommand | Purpose |
|---|---|
| `synci mode list` | List available research modes on this install. |
| `synci mode show` | Show the active mode for the current directory. |
| `synci mode set <name>` | Set the default mode for the current working directory. |

## Stats

### `synci stats`

Show credit usage by session, mode, and time window.

```bash
synci stats              # cumulative
synci stats --since 7d   # last 7 days
synci stats --json       # machine-readable
```

## Agent management

### `synci agent`

Manage agent profiles. Defaults, per-project overrides, and custom profiles.

| Subcommand | Purpose |
|---|---|
| `synci agent list` | Show available agent profiles. |
| `synci agent set-default <name>` | Set your default agent. |
| `synci agent show <name>` | Print the full configuration for an agent. |

## GitHub and PR helpers

### `synci github`

GitHub helpers that compose with `gh` and the Synci agent.

| Subcommand | Purpose |
|---|---|
| `synci github review <pr>` | Open a PR and ask the agent to review it. |
| `synci github resolve <issue>` | Open an issue and ask the agent to plan a fix. |

### `synci pr`

PR-focused workflows.

| Subcommand | Purpose |
|---|---|
| `synci pr open` | Create a PR from the current branch with an agent-generated body. |
| `synci pr update <pr>` | Update a PR description after new commits. |

## Sessions: export and import

### `synci export`

Export a session as a JSON document for sharing or archiving.

```bash
synci export <session-id> --out session.json
```

### `synci import`

Import a session JSON back into your account.

```bash
synci import session.json
```

Useful for handing off context to a teammate or restoring a session on a new machine.

## Distribution and lifecycle

### `synci upgrade`

Upgrade to the latest published version. Detects how you installed (npm, Homebrew, or direct binary) and runs the matching update flow.

```bash
synci upgrade
synci upgrade --version 1.2.0   # pin to a specific version
```

### `synci uninstall`

Remove the CLI and clear local config.

```bash
synci uninstall
```

The dashboard account, credentials, and session history are unaffected.

### `synci generate`

Generate boilerplate. The exact templates depend on the version, but typical generators include Synci CLI plugin scaffolds and project templates.

```bash
synci generate --help
```

## Server modes

### `synci serve`

Run the CLI as a long-lived backend server, exposing an HTTP API. Useful when you want a remote agent endpoint that other tools can call.

```bash
synci serve --port 4444
```

### `synci acp`

Speak the Agent Communication Protocol over stdio. This is what the desktop app and IDE plugins use to talk to the agent. You usually don't run this directly.

## Slash commands inside a session

Slash commands work inside any session: web UI, TUI, or desktop.

| Slash command | Action |
|---|---|
| `/mode <name>` | Switch research mode. |
| `/agent <name>` | Switch agents (also via the toggle in web/desktop, or <kbd>Tab</kbd> in TUI). |
| `/clear` | Clear the conversation context. |
| `/save` | Save the session to your dashboard. |
| `/export` | Export the conversation to a markdown file. |
| `/help` | Show available slash commands. |

## Environment variables

Useful env vars the CLI reads at startup:

| Variable | Purpose |
|---|---|
| `SYNSC_DASHBOARD_URL` | Override the dashboard URL (default `https://cli.syntheticsciences.ai`). |
| `SYNSC_DEFAULT_MODE` | Default research mode. |
| `SYNSC_DEFAULT_AGENT` | Default agent (`build` or `plan`). |

Provider keys for credentials (HF, W&B, Modal, Lambda, AWS, GCP) come from the dashboard sync. See [Credentials](/cli/credentials).
