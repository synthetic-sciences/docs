This walkthrough takes you from nothing installed to a productive Synsci CLI session in under five minutes. You'll run a single-command install, authenticate to your dashboard, and open the recommended browser UI with `synsc web`.

<Steps>
  <Step title="Install in one command">
    Run the all-in-one installer:

    ```bash
    npx synsci
    ```

    This downloads the platform binary, walks you through dashboard authentication, and pre-fetches your synced credentials. When it finishes, `synsc` is on your PATH and ready.

    See [Installation](/cli/installation) if you'd rather use npm, Homebrew, or curl directly.
  </Step>

  <Step title="Verify the connection">
    Confirm the install authenticated cleanly:

    ```bash
    synsc connect status
    ```

    Healthy output prints your account email, current credit balance, and the synced credentials available — Hugging Face, Weights & Biases, Modal, Lambda, AWS, and GCP all show up if you have them set in the dashboard.

    <Tip>
      Add or rotate credentials at [cli.syntheticsciences.ai](https://cli.syntheticsciences.ai). The CLI picks them up on the next session start.
    </Tip>
  </Step>

  <Step title="Open the web UI">
    The recommended way to use Synsci CLI day-to-day is the browser UI:

    ```bash
    synsc web
    ```

    `synsc web` boots a local server and opens your default browser. You get a clean chat interface, multi-session tabs, side-by-side file diffs, and the same agent surface as the TUI — without giving up a terminal split. See [Web UI](/cli/web) for the full feature set.
  </Step>

  <Step title="Send your first prompt">
    In the web UI, type a prompt and press Enter:

    ```
    Read README.md and summarize what this repo does in three bullets.
    ```

    The agent reads the file, streams its response, and prompts you for the next message. You're in the **build** agent on the default model (Claude Sonnet 4.5).
  </Step>

  <Step title="Switch agents and models">
    Two top-level agents are available:

    - **build** — full filesystem and shell access (default).
    - **plan** — read-only mode for analysis without mutations.

    Switch agents with the agent toggle in the UI header (or <kbd>Tab</kbd> in the TUI). Switch models from the model picker — your subscription includes Claude Opus 4, Claude Sonnet 4.5, GPT Codex Mini, and Gemini 2.5 Pro.

    Mention `@general` in a message to delegate a search or multi-step task to a sub-agent that runs in its own context window:

    ```
    @general find every place we call the OpenAI API and list the call sites
    ```
  </Step>

  <Step title="Inspect usage">
    After running a few prompts, check what you spent:

    ```bash
    synsc stats
    ```

    `stats` aggregates token usage and credit cost per model, per session, and across time windows. The full audit trail and billing live on the [dashboard](https://cli.syntheticsciences.ai).
  </Step>

  <Step title="Run one-shot prompts (optional)">
    For scripted workflows, skip the UI and pipe a single prompt with `synsc run`:

    ```bash
    synsc run "Refactor utils/log.ts to use structured logging"
    ```

    The agent runs the prompt to completion and exits — useful for git hooks, CI, and shell pipelines.
  </Step>
</Steps>
